5. **Create `backend/src/feed/feed.service.ts`:**
   Core feed engine with these methods:

   **`getFeed(params)`** - Main feed query:
   - Parameters: `{ lat, lng, radiusMiles (default 10), first (default 20), after?, filters?, lastFetchedAt? }`
   - Uses `prisma.$queryRaw` with PostGIS `ST_DWithin` for radius filtering
   - 7-day rolling window: `AND "scrapedAt" > NOW() - INTERVAL '7 days'`
   - Applies filter WHERE clauses (cuisine, meal, dietary)
   - Orders by `velocityScore DESC, id ASC`
   - Cursor pagination: decode cursor to `{ id, velocity }`, use keyset pagination `AND ("velocityScore" < cursor.velocity OR ("velocityScore" = cursor.velocity AND id > cursor.id))`
   - Fetch `first + 1` results to determine hasNextPage
   - Compute engagementHumanized per recipe using `VelocityScorer.humanize()`
   - If `lastFetchedAt` provided, count `newSinceLastFetch` (recipes with `scrapedAt > lastFetchedAt` in radius)
   - Returns `RecipeConnection` with edges, pageInfo, totalCount, lastRefreshed

   **CRITICAL PostGIS query structure:**
   ```sql
   SELECT id, name, "imageUrl", "imageStatus", "prepTime", calories,
          "engagementLoves", "isViral", "cuisineType", "mealType", "velocityScore",
          "scrapedAt",
          ST_Distance(
            ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)::geography,
            ST_SetSRID(ST_MakePoint($userLng, $userLat), 4326)::geography
          ) / 1609.34 as "distanceMiles"
   FROM "Recipe"
   WHERE ST_DWithin(
     ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)::geography,
     ST_SetSRID(ST_MakePoint($userLng, $userLat), 4326)::geography,
     $radiusMeters
   )
   AND latitude IS NOT NULL AND longitude IS NOT NULL
   AND "scrapedAt" > NOW() - INTERVAL '7 days'
   ${filterConditions}
   ${cursorCondition}
   ORDER BY "velocityScore" DESC, id ASC
   LIMIT ${first + 1}
   ```
   Note: ST_MakePoint(longitude, latitude) - LONGITUDE FIRST!

   **`getFeedWithFallback(params)`** - Wraps getFeed with expanded radius fallback:
   - Call getFeed with user's radius
   - If zero results, expand to 50 miles (city-level)
   - If still zero, expand to 500 miles (country-level), set `expandedFrom: 'city', expandedTo: 'country'`
   - If still zero, query all recipes (global), set `expandedTo: 'global'`
   - Per user decision: "Never return empty feed"

   **`getFeedWithFilterRelaxation(params)`** - Handles zero-result filter queries:
   - If filters produce zero results in radius, progressively relax:
     1. Drop dietaryTags first (most restrictive)
     2. Drop mealTypes
     3. Drop cuisineTypes
   - Set `partialMatch: true` and `filtersRelaxed: ['gluten-free']` on response
   - Per user decision: "When filters produce zero results: return nearest partial matches"

   **`buildFilterClause(filters)`** - Private helper:
   - CuisineTypes: `"cuisineType" = ANY(ARRAY[...]::\"CuisineType\"[])` (OR within)
   - MealTypes: `"mealType" = ANY(ARRAY[...]::\"MealType\"[])` (OR within)
   - DietaryTags: `"dietaryTags" @> ARRAY[...]::text[]` (AND logic - must have ALL tags)
   - Combine across categories with AND

   **Cursor encoding/decoding:**
   - `encodeCursor({ id, velocity }): string` - `Buffer.from(JSON.stringify({id, velocity})).toString('base64')`
   - `decodeCursor(cursor: string): { id: string; velocity: number }` - reverse

   **IMPORTANT edge cases:**
   - Recipes with null lat/lng must be filtered out: `AND latitude IS NOT NULL AND longitude IS NOT NULL`
   - Coordinate validation: reject lat outside [-90, 90] or lng outside [-180, 180]
   - If PostGIS extension is not available, fall back to Prisma `findMany` with basic filtering (degrade gracefully, log warning)
