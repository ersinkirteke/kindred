1. Create backend/src/voice/dto/narration-request.dto.ts:
   - NarrationMetadataDto (GraphQL ObjectType): speakerName, relationship, recipeId, recipeName
   - This DTO is returned alongside the streaming audio URL so mobile clients can display "Narrated by Mom"

2. Create backend/src/voice/narration.service.ts:
   - @Injectable() service
   - Inject: PrismaService, ElevenLabsService, GoogleGenerativeAI (same pattern as RecipeParserService)
   - Initialize Gemini model: gemini-2.0-flash

   Methods:
   a) rewriteToConversational(recipe: { name, description, steps, ingredients }): Promise<string>
      - First check cache: query NarrationScript table for recipeId
      - If cached, return cached conversationalText
      - If not cached, call Gemini with prompt:
        ```
        Rewrite this recipe into warm, conversational narration as if a loved one is guiding you through cooking.

        Recipe: {recipe.name}
        Description: {recipe.description}

        Ingredients:
        {recipe.ingredients.map(i => `- ${i.name}: ${i.amount} ${i.unit}`).join('\n')}

        Steps:
        {recipe.steps.sort(orderIndex).map((s, i) => `${i + 1}. ${s.text}`).join('\n')}

        Guidelines:
        - Start with warm intro: "Today we're making {recipe name}"
        - List ingredients overview: "You'll need {ingredients}"
        - Rewrite each step conversationally: "Chop onions" → "Now, let's chop up those onions"
        - Add natural pauses between steps (insert "[PAUSE]" markers for 2-second gaps)
        - Keep it warm and encouraging, like a loved one talking you through it
        - Total length: suitable for 2-5 minutes when spoken aloud

        Output only the narration script, no explanations.
        ```
      - Cache result: upsert NarrationScript { recipeId, conversationalText, generatedAt }
      - Return conversational text

   b) streamRecipeNarration(recipeId: string, voiceProfileId: string, userId: string, response: Express.Response): Promise<void>
      - Verify voice profile ownership and status=READY
      - Get voice profile (need elevenLabsVoiceId and speakerName for metadata)
      - Load recipe with steps and ingredients
      - Call rewriteToConversational(recipe) to get narration text
      - Replace [PAUSE] markers with "..." (natural pause in TTS)
      - Set response headers:
        Content-Type: audio/mpeg
        Transfer-Encoding: chunked
        Cache-Control: no-cache
        X-Speaker-Name: {voiceProfile.speakerName} (metadata header for mobile client)
        X-Speaker-Relationship: {voiceProfile.relationship}
        X-Recipe-Name: {recipe.name}
      - Call elevenLabsService.generateSpeechStream(elevenLabsVoiceId, narrationText)
      - Pipe ReadableStream chunks to response.write()
      - Call response.end() when stream completes
      - Handle errors: if ElevenLabs stream fails mid-way, end response gracefully

   c) getNarrationMetadata(recipeId: string, voiceProfileId: string, userId: string): Promise<NarrationMetadataDto>
      - Return metadata without streaming audio (for UI display before pressing play)
      - Verify voice profile ownership
      - Return: speakerName, relationship, recipeId, recipeName

3. Add NarrationScript model to Prisma schema (cache table):
   - id: String @id @default(cuid())
   - recipeId: String @unique (one cached narration per recipe — voice-independent)
   - conversationalText: String (the rewritten narration)
   - generatedAt: DateTime @default(now())
   - @@index([recipeId])

   Run `npx prisma db push` after adding model.
