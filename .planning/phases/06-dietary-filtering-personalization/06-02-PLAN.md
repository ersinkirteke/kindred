    **Step 1: AffinityScore model** (AffinityScore.swift):
    - Simple struct: `public struct AffinityScore: Equatable, Identifiable { public let cuisineType: String; public let score: Double; public var id: String { cuisineType } }`.
    - Score range: 0.0 to 1.0 (normalized).
    - Conforms to Identifiable for SwiftUI ForEach rendering.

    **Step 2: CulinaryDNAEngine** (CulinaryDNAEngine.swift):
    - Class with computation logic. Does NOT need @MainActor — receives data as parameters, doesn't access SwiftData directly.
    - Constants:
      - `bookmarkWeight: Double = 2.0` (bookmarks are 2x stronger per locked decision)
      - `skipWeight: Double = 1.0`
      - `decayHalfLife: TimeInterval = 30 * 24 * 60 * 60` (30 days)
      - `activationThreshold: Int = 50` (per PERS-01 requirement)
    - Method `computeAffinities(bookmarks: [GuestBookmark], skips: [GuestSkip]) -> [AffinityScore]`:
      1. Build `cuisineScores: [String: Double]` dictionary.
      2. For each bookmark with non-nil cuisineType: `cuisineScores[cuisine] += bookmarkWeight * exponentialDecay(age)`.
      3. For each skip with non-nil cuisineType: `cuisineScores[cuisine] -= (skipWeight * exponentialDecay(age)) / 5.0`. The `/5.0` means it takes ~5 skips to cancel 1 bookmark (per locked decision: "takes 5-10 skips of same cuisine to noticeably reduce affinity").
      4. Find max positive score. If max <= 0, return empty array.
      5. Normalize all scores to 0.0-1.0 range: `score / maxScore`, clamp to `max(0, ...)`.
      6. Sort descending by score, return as `[AffinityScore]`.
    - Method `exponentialDecay(age: TimeInterval) -> Double`:
      - `max(0.001, pow(0.5, age / decayHalfLife))` — clamp to prevent underflow for very old interactions (per pitfall 8 from research).
    - Method `interactionCount(bookmarks: [GuestBookmark], skips: [GuestSkip]) -> Int`:
      - Returns `bookmarks.count + skips.count`.
    - Method `isActivated(bookmarks: [GuestBookmark], skips: [GuestSkip]) -> Bool`:
      - Returns `interactionCount >= activationThreshold`.

    **Step 3: FeedRanker** (FeedRanker.swift):
    - Struct with re-ranking logic.
    - Constants: `personalizedRatio: Double = 0.6`, `discoveryRatio: Double = 0.4` (per locked decision: ~60% preferred, ~40% discovery).
    - Method `rerank(recipes: [RecipeCard], affinities: [AffinityScore]) -> [RecipeCard]`:
      1. Guard `!affinities.isEmpty` else return recipes unchanged.
      2. Build `affinityMap: [String: Double]` from affinities array.
      3. For each recipe, compute combined score: `(personalizedRatio * affinityScore) + (discoveryRatio * normalizedVelocity)`.
        - `affinityScore` = `affinityMap[recipe.cuisineType] ?? 0.0`.
        - `normalizedVelocity` = `recipe.velocityScore / maxVelocity` (normalize velocity to 0-1 range). If maxVelocity is 0, use 1.0.
      4. Sort by combined score descending.
      5. Return sorted recipes.
    - Method `isPersonalized(recipe: RecipeCard, affinities: [AffinityScore]) -> Bool`:
      - Returns true if recipe's cuisineType is in top 3 affinities (per research recommendation: "For You" badge when cuisine is in top 3).
      - This determines which recipes get the "For You" badge.

      - `isPersonalized` returns false.
    - Register in `DependencyValues` extension: `var personalizationClient: PersonalizationClient`.
