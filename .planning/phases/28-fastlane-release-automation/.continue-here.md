**If sigh fails**, fallback is manual creation at https://developer.apple.com/account/resources/profiles/add — pick "App Store", app ID `com.ersinkirteke.kindred`, select the Apple Distribution cert we just created, download the `.mobileprovision`, double-click to install.

### Step 2: Fix the Fastfile export_options

Read `Kindred/fastlane/Fastfile` around line 97 (the `build_app` call). One of the Wave 2 sub-agent's auto-fixes set `export_options` to something like:
```ruby

  provisioningProfiles: {},
  signingStyle: "manual",
  teamID: "CV9G42QVG4",
```

Change to **automatic signing** (simplest fix):
```ruby
export_options: {
  signingStyle: "automatic",
  teamID: "CV9G42QVG4",

```

OR manual signing with explicit profile mapping (more control):
```ruby
export_options: {
  signingStyle: "manual",

  method: "app-store",
  provisioningProfiles: {
    "com.ersinkirteke.kindred" => "<profile name from sigh output>"
  }
}
```

Note: Xcode 15+ prefers `method: "app-store-connect"` instead of `"app-store"` (deprecation warning seen in log). Update if you want to silence it.
