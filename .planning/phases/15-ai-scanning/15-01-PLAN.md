<objective>
Build the backend AI analysis pipeline for fridge photo recognition and receipt text parsing using Gemini 2.0 Flash.

Purpose: Enable the server to analyze uploaded fridge photos via Gemini Vision API and parse receipt OCR text into structured ingredient lists with confidence scores, categories, storage locations, and expiry estimates. This is the intelligence layer that transforms raw photos/text into actionable pantry data.

Output: ScanAnalyzerService with two Gemini prompts (fridge vision + receipt text), two new GraphQL mutations (analyzeScan, analyzeReceiptText), Prisma ScanJob model for result persistence, and server-side free scan quota tracking.
</objective>

**4. Create scan-analyzer.service.ts:**

Follow RecipeParserService pattern exactly:
- Import GoogleGenerativeAI from @google/generative-ai
- Get GOOGLE_AI_API_KEY from ConfigService
- Initialize model with gemini-2.0-flash-exp, temperature 0.1, responseMimeType 'application/json'
- Handle missing API key gracefully (warn log, return empty results)

Two methods:

`analyzeFridgePhoto(photoUrl: string): Promise<DetectedItemDto[]>`
- Fetch image from R2 URL via fetch()
- Convert to base64
- Send to Gemini with fridge analysis prompt:
  - Only list clearly visible items (no guessing at hidden items) -- per user decision
  - Normalize names to English (Turkish product names to English) -- per user decision
  - Categories MUST be one of: dairy, produce, meat, seafood, grains, baking, spices, beverages, snacks, condiments
  - Estimate quantities where possible ("~6 eggs", "~500g chicken") -- per user decision
  - Storage location: fridge, freezer, or pantry based on item type -- per user decision
  - Expiry estimate: conservative days from today -- per user decision
  - Confidence: 0-100 score
  - Return ONLY the JSON array
- Parse JSON response into DetectedItemDto[]
- Wrap in try/catch, log errors, throw descriptive error on failure
- 30-second timeout via AbortController on the Gemini call

`analyzeReceiptText(ocrText: string): Promise<DetectedItemDto[]>`
- Send text-only prompt (no image) to Gemini
- Prompt optimized for receipt parsing:
  - Expand abbreviations (e.g., "K FL MW" -> "Kellogg's Froot Loops")
  - Normalize to English ingredient names (ignore brand names)
  - Only extract FOOD items (skip toiletries, household goods)
  - Same categories, storage location, expiry estimates as fridge prompt
  - Lower confidence for ambiguous abbreviations
- Parse JSON response into DetectedItemDto[]
- Same error handling pattern
