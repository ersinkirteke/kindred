    **2. Create `SearchResultsView.swift`** (NEW FILE in FeedFeature/Sources/Feed/):

    A scrollable vertical list that displays search results with pagination:
    ```swift
    struct SearchResultsView: View {
        let store: StoreOf<FeedReducer>
        let heroNamespace: Namespace.ID

        var body: some View {
            ScrollView {
                LazyVStack(spacing: 16) {
                    ForEach(store.searchResults) { recipe in
                        SearchResultCardView(
                            recipe: recipe,
                            heroNamespace: heroNamespace,
                            onTap: { id in store.send(.openRecipeDetail(id)) }
                        )
                        .onAppear {
                            // Auto-load more when within 3 cards of bottom
                            if recipe == store.searchResults.suffix(3).first {
                                store.send(.loadMoreSearchResults)
                            }
                        }
                    }

                    // Bottom loading spinner for pagination
                    if store.isSearching && !store.searchResults.isEmpty {
                        ProgressView()
                            .padding()
                    }
                }
                .padding(.horizontal, 16)
            }
            .scrollDismissesKeyboard(.interactively)
        }
    }
    ```

    **3. Update `FeedView.swift`:**

    Add a sticky search bar above DietaryChipBar, below the location pill toolbar. The search bar and chips stay visible always; only the content area below switches.

    a) Add a `TextField` search bar with magnifying glass icon and X clear button:
       - Bound to `store.searchQuery` via `store.send(.searchQueryChanged(newValue))`
       - X button calls `store.send(.clearSearch)`
       - Disabled + grayed when `store.isQuotaExhausted`
       - Placeholder: "Search recipes..." (localized)
       - Style: rounded rect background, magnifying glass leading icon, X trailing icon when text non-empty

    b) Add result count text below DietaryChipBar when in search mode:
       - When `store.feedMode == .search && !store.searchResults.isEmpty`:
         Build descriptive string like "12 recipes found" or "12 vegan recipes found" (include active filter names if any)
       - Use `.font(.footnote)` + `.foregroundStyle(.secondary)` for subtle display
       - Also show `store.searchTotalCount` if available

    c) Switch content area based on `store.feedMode`:
       - `.browse`: existing card stack + swipe UI (unchanged)
       - `.search`:
         - If `store.isSearching && store.searchResults.isEmpty`: show centered `ProgressView` (inline spinner)
         - Else if `store.searchResults.isEmpty && !store.isSearching && store.searchQuery.count >= 3`:
           Show `EmptyStateView` with message "No recipes found for '\(store.searchQuery)'. Try a broader search or remove filters."
           If `store.activeDietaryFilters` is non-empty, add a "Clear Filters" button below that calls `store.send(.dietaryFilterChanged([]))`
         - Else if `store.isQuotaExhausted`:
           Show message "Recipe search is temporarily unavailable. Browse popular recipes instead." with a button to clear search
         - Else if `store.searchQuery.count < 3 && store.searchQuery.count > 0`:
           Show nothing (or the browse feed underneath) — debounce hint is silent per CONTEXT.md
         - Else: show `SearchResultsView(store: store, heroNamespace: heroNamespace)`

    d) When offline (`store.isOffline`) and in search mode: show "Search requires internet" message. Keep the search bar but show the message in the content area.

    e) Animation: use `.animation(.easeInOut(duration: 0.2), value: store.feedMode)` for smooth transitions.
