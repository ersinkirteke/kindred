    1. Update `startMigration` handler: set `pendingMigration` UserDefaults flag before calling client.

    2. Update `migrationSucceeded` handler:
       ```swift

           state.isMigrating = false
           state.migrationRetryCount = 0
           // Clear pending migration flag
           return .run { _ in
               UserDefaults.standard.removeObject(forKey: "pendingMigration")
           }
       ```

    3. Update `migrationFailed` handler: after max retries exceeded, do NOT clear `pendingMigration` flag — leave it so migration retries on next app launch.

    4. In `.connectivityChanged` handler, when coming back online (`wasOffline && !isOffline`), check for pending migration:

       ```swift
           var effects: [Effect<Action>] = [.send(.feed(.refreshFeed))]

           // Retry pending migration when connectivity returns
           if UserDefaults.standard.bool(forKey: "pendingMigration"),
              !state.isMigrating,
              state.migrationRetryCount < 3 {
               effects.append(.send(.startMigration))
           }

           return .merge(effects)
       }
       ```

    5. In `.observeAuth` or app startup path, check for pending migration on launch:
       Add a new action `case checkPendingMigration` that checks the flag and triggers migration if needed:

       ```swift
       case .checkPendingMigration:
           if UserDefaults.standard.bool(forKey: "pendingMigration"),
              case .authenticated = state.currentAuthState,
              !state.isMigrating {
               return .send(.startMigration)
           }
           return .none
       ```
       Trigger this from `.authStateChanged` when user is already authenticated on app launch (not just first sign-in).

    6. Handle the updated `MigrationResult` return type. Update `.startMigration` and `.retryMigration`:

       ```swift
       case .startMigration:

           state.migrationRetryCount = 0

               } catch {
                   Logger.migration.error("Failed: \(error.localizedDescription)")
                   await send(.migrationFailed)
               }
           }
       ```
       Same pattern for `.retryMigration`.
