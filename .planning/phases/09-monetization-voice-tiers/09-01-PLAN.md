    Create SubscriptionReducer.swift — TCA @Reducer following VoicePlaybackReducer pattern:
    - State (all @ObservableState):
      - `subscriptionStatus: SubscriptionStatus = .unknown`
      - `products: [Product] = []`
      - `isPurchasing: Bool = false`
      - `isRestoring: Bool = false`
      - `showPaywall: Bool = false`
      - `error: String?`
      - `displayPrice: String = "$9.99"` (fallback, updated from Product.displayPrice)
    - Actions:
      - `onAppear` — loads products + checks current entitlement
      - `productsLoaded([Product])` — stores products, extracts displayPrice
      - `entitlementChecked(SubscriptionStatus)` — updates subscriptionStatus
      - `subscribeTapped` — initiates purchase flow
      - `purchaseCompleted(SubscriptionStatus, StoreKit.Transaction)` — updates status, hides paywall, syncs JWS to backend
      - `purchaseFailed(String)` — sets error
      - `restoreTapped` — calls restorePurchases, re-checks entitlement
      - `restoreCompleted(SubscriptionStatus)` — updates status
      - `showPaywall` / `dismissPaywall` — toggles showPaywall
      - `transactionUpdated(SubscriptionStatus)` — from Transaction.updates stream
      - `clearError`
    - Effects:
      - `onAppear`: parallel load products + check entitlement + start transaction updates stream
      - `subscribeTapped`: set isPurchasing=true, call `subscriptionClient.purchase(product)` on first product. On success, send `.purchaseCompleted(entitlementStatus, transaction)` — MUST pass the `StoreKit.Transaction` returned by `purchase()` so that `purchaseCompleted` can extract JWS and sync to backend. On failure, send `.purchaseFailed(error.localizedDescription)`.
      - `purchaseCompleted`: After updating subscription status and hiding paywall, sync JWS to backend. The effect body MUST include this wiring:
        ```swift
        case .purchaseCompleted(let status, let transaction):
          state.subscriptionStatus = status
          state.isPurchasing = false
          state.showPaywall = false
          return .run { _ in
            // Sync purchase to backend — fire-and-forget (don't block UI)
            if let jws = await subscriptionClient.jwsRepresentation(transaction) {
              _ = try? await subscriptionClient.syncSubscriptionToBackend(jws)
            }
          }
        ```
        This calls `subscriptionClient.syncSubscriptionToBackend(jws)` which hits the backend `verifySubscription` GraphQL mutation (created in Plan 04). Without this wiring, the backend has no subscription record and will reject voice profile creation for Pro users (VOICE-07).
        IMPORTANT: The `purchaseCompleted` action must carry the `StoreKit.Transaction` alongside the `SubscriptionStatus` so the JWS can be extracted. Update the action signature to `case purchaseCompleted(SubscriptionStatus, StoreKit.Transaction)`.
        Error handling: use `try?` — don't surface backend sync failures to the user. The purchase is already confirmed by Apple; backend sync is best-effort and will be retried on next app launch via Transaction.updates.
      - `restoreTapped`: set isRestoring=true, call restorePurchases, re-check entitlement, send restoreCompleted
