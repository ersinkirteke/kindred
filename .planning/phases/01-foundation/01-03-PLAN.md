Build the recipe scraping pipeline that discovers trending recipes from X API (and Instagram partner placeholder), parses them into structured data using Gemini AI, and stores them in PostgreSQL. Purpose: INFR-02 requires trending recipe discovery from social platforms by location. INFR-04 requires graceful degradation when sources are unavailable. This pipeline feeds the entire app -- without recipes, there is no feed.

    1. Create `backend/src/scraping/scraping.service.ts`:
       - Inject XApiService, InstagramService, RecipeParserService, PrismaService, and a Logger
       - `scrapeForCity(city: string): Promise<{ newRecipes: number; duplicates: number; parseFailures: number }>`:
         a. Fetch raw posts from X API + Instagram (combine results)
         b. Deduplicate by sourceId: check Prisma for existing recipe with same sourceId, skip if exists
         c. Parse each raw post through RecipeParserService
         d. For successfully parsed recipes, create in DB via Prisma:
            - Set recipe fields from ParsedRecipe
            - Set scrapedFrom (platform), sourceId, location (city), scrapedAt (now)
            - Set imageStatus to PENDING (Plan 04 will generate images)
            - Create nested Ingredient and RecipeStep records with orderIndex
            - Calculate isViral flag: engagementCount > threshold (configurable, default 1000)
         e. Log summary: X new, Y duplicates, Z parse failures
         f. Return counts for monitoring
       - `scrapeWithFallback(city: string): Promise<void>`:
         a. Try scrapeForCity(city) -- if returns 0 new recipes AND 0 cached for city:
         b. Expand to country level: scrapeForCity(country derived from city, e.g., "United States")
         c. If still 0: scrapeForCity("global") -- trending worldwide
         d. Log each fallback expansion step
         This implements INFR-04: "expand radius (city -> country -> global trending)"
       - Private helper: `deriveCountryFromCity(city: string): string` -- simple mapping, can be enhanced later

    2. Create `backend/src/scraping/scraping.scheduler.ts`:
       - Inject ScrapingService and ConfigService
       - Use @nestjs/schedule decorators
       - Default target cities: configurable via SCRAPING_TARGET_CITIES env var (comma-separated), default to "New York,Los Angeles,Chicago,Houston,Phoenix"
       - Schedule 4x/day at fixed times using @Cron:
         - 8:00 AM UTC: `@Cron('0 8 * * *')`
         - 12:00 PM UTC: `@Cron('0 12 * * *')`
         - 6:00 PM UTC: `@Cron('0 18 * * *')`
         - 9:00 PM UTC: `@Cron('0 21 * * *')`
       - Each cron job iterates target cities, calls scrapeWithFallback for each
       - Add manual trigger method: `triggerScrape(city?: string)` for admin/testing use
       - Log start/end of each scrape cycle with timing
