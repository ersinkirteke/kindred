  <action>
    1. Install dependency: `npm install firebase-admin` (if not already installed from Plan 01).

    2. Create `backend/src/push/dto/register-device.input.ts`:
       - @InputType RegisterDeviceInput with:
         - token: String (required, the FCM device token)
         - platform: Platform enum (IOS or ANDROID, reuse from GraphQL models)
       - @InputType SendNotificationInput (for admin/testing):
         - userId: String (target user)
         - title: String
         - body: String
         - data: JSON (optional custom payload)

    3. Create `backend/src/push/push.service.ts`:
       - Inject ConfigService, PrismaService, Logger
       - On module init: initialize firebase-admin app with service account from FIREBASE_SERVICE_ACCOUNT_PATH. If env var not set, log warning and disable push (do not crash -- allows local dev without Firebase).
       - `registerDeviceToken(userId: string, token: string, platform: 'IOS' | 'ANDROID'): Promise<DeviceToken>`:
         - Upsert DeviceToken in DB (update if token exists, create if new)
         - One user can have multiple devices (phone + tablet)
       - `removeDeviceToken(token: string): Promise<void>`:
         - Delete token from DB (called when token becomes invalid)
       - `sendToUser(userId: string, notification: { title: string; body: string; data?: Record<string, string> }): Promise<{ success: number; failures: number }>`:
         - Find all device tokens for userId
         - Send via firebase-admin messaging().sendEachForMulticast (handles both iOS/Android)
         - For iOS: include APNs headers with priority 10 and sound 'default'
         - For Android: include FCM notification channel
         - Handle invalid token errors: if firebase returns NotRegistered or InvalidRegistration, remove token from DB (device uninstalled app)
         - Return success/failure counts
       - `sendToMultipleUsers(userIds: string[], notification: { title: string; body: string; data?: Record<string, string> }): Promise<void>`:
         - Batch send to multiple users (for future expiry alerts, engagement nudges)
         - Process in batches of 500 (FCM multicast limit)
       - `sendTestNotification(userId: string): Promise<boolean>`:
         - Send test notification with title "Kindred" and body "Push notifications are working!" for verification

    4. Create `backend/src/push/device-token.resolver.ts`:
       - @Resolver()
       - @Mutation `registerDevice(input: RegisterDeviceInput): DeviceToken` -- protected with @UseGuards(ClerkAuthGuard), uses @CurrentUser to get userId, calls PushService.registerDeviceToken
       - @Mutation `unregisterDevice(token: String!): Boolean` -- protected, removes device token
       - Import ClerkAuthGuard from auth module (available via Plan 02 dependency)

    5. Create `backend/src/push/push.module.ts`:
       - Import PrismaModule, ConfigModule, AuthModule
       - Provide PushService, DeviceTokenResolver
       - Export PushService (other modules will use it for sending notifications)

    6. Update `backend/src/app.module.ts`:
       - Import PushModule

    7. Add to `backend/.env.example`:
       - FIREBASE_SERVICE_ACCOUNT_PATH=./firebase-service-account.json
  </action>

  </verify>
  <done>PushService sends notifications via FCM with platform-specific payloads. Device tokens managed via GraphQL mutations. Invalid tokens auto-cleaned. Firebase initialization is optional (dev works without it). All compiles.</done>
