       - Define ClerkWebhookPayload interface matching Clerk's webhook format:
         `{ data: { id: string; email_addresses: Array<{ email_address: string }>; first_name?: string; last_name?: string; image_url?: string }; type: 'user.created' | 'user.updated' | 'user.deleted' }`

    2. Create `backend/src/auth/clerk-webhook.controller.ts`:
       - This is a REST controller (not GraphQL) since Clerk sends HTTP POST webhooks
       - @Controller('webhooks') with @Post('clerk') endpoint
       - CRITICAL: Verify webhook signature using svix library and CLERK_WEBHOOK_SECRET env var. Reject unsigned requests with 401. This prevents attackers from forging user events.
       - On 'user.created': call UsersService.upsertFromClerk(clerkId, email, displayName)
       - On 'user.updated': call UsersService.upsertFromClerk to update email/name
       - On 'user.deleted': call UsersService.softDelete(clerkId) or mark user inactive
       - Return 200 on success, 400 on invalid payload, 401 on bad signature
       - Use raw body for signature verification (not parsed JSON) -- configure NestJS rawBody option

    3. Update `backend/src/users/users.service.ts`:
       - Add `upsertFromClerk(clerkId: string, email: string, displayName?: string)`: Use Prisma upsert -- create if not exists, update email/displayName if exists. Return User.
       - Add `findByClerkId(clerkId: string)`: Find user by Clerk ID, return null if not found.
       - Keep existing findById method.

    4. Update `backend/src/users/users.resolver.ts`:
       - Add @UseGuards(ClerkAuthGuard) to `me` query
       - `me` query uses @CurrentUser() to get clerkId, calls UsersService.findByClerkId, returns User or throws NotFoundException
       - Add @Query `myBookmarks` (guarded): returns user's bookmarked recipes via Prisma include

    5. Update `backend/src/auth/auth.module.ts`:
       - Import UsersModule (for webhook controller to use UsersService)
       - Add ClerkWebhookController to controllers array
