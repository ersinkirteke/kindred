**AVSpeechClient.swift:**

- `speak: @Sendable ([String], PlaybackSpeed) async throws -> Void` — accepts preprocessed step strings and initial speed

**AVSpeechManager.swift:**
Create `@MainActor final class AVSpeechManager: NSObject, AVSpeechSynthesizerDelegate`:
- `static let shared = AVSpeechManager()`
- Stored properties: `synthesizer: AVSpeechSynthesizer?`, `steps: [String]`, `currentStepIndex: Int`, `speed: Float`, `retryCount: Int`, `timeoutTask: Task<Void, Never>?`
- AsyncStream continuations: `statusContinuation`, `stepContinuation`

Key methods:
- `speak(steps:speed:)` — stores steps, resets index/retry, calls `enqueueUtterances(from: 0)`
- `enqueueUtterances(from:)` — deallocates old synthesizer, creates new `AVSpeechSynthesizer()`, sets delegate, sets `usesApplicationAudioSession = true`, iterates steps from index creating `AVSpeechUtterance` with `preferredVoice()`, mapped rate, `postUtteranceDelay = 1.2`, then calls `synthesizer?.speak()` for each. Starts 5-second silent failure timeout.
- `pause()` — `synthesizer?.pauseSpeaking(at: .word)`
- `resume()` — `synthesizer?.continueSpeaking()`
- `stopSpeaking()` — `synthesizer?.stopSpeaking(at: .immediate)`, cancel timeout
- `jumpToStep(index:)` — stop current, update `currentStepIndex`, re-enqueue from index
- `setRate(rate:)` — store new rate. Note: AVSpeechSynthesizer doesn't support changing rate mid-utterance. Rate applies to next utterance enqueued. If currently speaking, stop and re-enqueue from current step with new rate.
- `statusStream()` / `stepIndexStream()` — create and return AsyncStreams, store continuations
- `cleanup()` — stopSpeaking, nil delegate, nil synthesizer, cancel timeout, deactivate audio session with `.notifyOthersOnDeactivation`, finish continuations

```swift
    return 0.35 + normalized * 0.30
```

Create `TextPreprocessor.swift` in the AVSpeech/ directory.

Implement as an enum with static methods (no instances needed). The `prepare(_ rawText: String, stepNumber: Int) -> String` method runs the full pipeline in this order (per CONTEXT.md locked decision):
1. Strip HTML tags — regex `<[^>]+>` replacement with empty string. Fast-path: skip if no `<` in text.
2. Strip markdown formatting — `**bold**` → `bold`, `*italic*` → content, `# headers` → content, `- bullets` → content
3. Expand Unicode fractions — dictionary lookup: `½` → `one half`, `¼` → `one quarter`, `¾` → `three quarters`, `⅓` → `one third`, `⅔` → `two thirds`
4. Expand cooking abbreviations — word-boundary regex (case-insensitive):
   - `\btbsp\.?\b` → `tablespoon`
   - `\btsp\.?\b` → `teaspoon`
   - `\boz\.?\b` → `ounce`
   - `\blb\.?\b` → `pound`
   - `\blbs\.?\b` → `pounds`
   - `\bml\.?\b` → `milliliter`
   - `\bg\.?\b(?=\s)` → `gram` (lookahead to avoid matching "g" at end of words)
   - `\bkg\.?\b` → `kilogram`
   - `\bmin\.?\b` → `minute` (careful: only when in cooking context — this is acceptable for English recipes per CONTEXT.md)
5. Normalize amounts — `(\d+)-(\d+)` → `$1 to $2`, `~(\d+)` → `approximately $1`
6. Prefix with step number — `"Step \(stepNumber): \(text)"` (per CONTEXT.md locked decision)

Use `static let` for compiled regex patterns where possible for performance (avoid recompiling on each call). All patterns use `.regularExpression` and `.caseInsensitive` options on `replacingOccurrences`.

English-only abbreviation expansion per CONTEXT.md decision. No SSML.

Also add a convenience method:
```swift
static func prepareSteps(_ steps: [String]) -> [String] {
    steps.enumerated().map { index, text in
        prepare(text, stepNumber: index + 1)
    }
}
```
