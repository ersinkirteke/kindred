    **Create SpoonacularBatchScheduler** (backend/src/spoonacular/spoonacular-batch.scheduler.ts):

    ```typescript
    @Injectable()
    export class SpoonacularBatchScheduler {
      private readonly logger = new Logger(SpoonacularBatchScheduler.name);
      private lastSuccessfulRun: Date | null = null;

      constructor(
        private spoonacularService: SpoonacularService,
        private cacheService: SpoonacularCacheService,
        private prisma: PrismaService,
      ) {}

      @Cron('0 2 * * *', { name: 'prewarm-recipes', timeZone: 'UTC' })
      async prewarmRecipes() { await this.executePrewarm('2:00 AM UTC'); }

      @Cron('0 3 * * *', { name: 'prewarm-retry-1', timeZone: 'UTC' })
      async retryPrewarm1() {
        if (this.shouldRetry(1)) {
          this.logger.warn('2 AM job failed, retrying at 3 AM');
          await this.executePrewarm('3:00 AM UTC (retry 1)');
        }
      }

      @Cron('0 4 * * *', { name: 'prewarm-retry-2', timeZone: 'UTC' })
      async retryPrewarm2() {
        if (this.shouldRetry(2)) {
          this.logger.error('2 AM and 3 AM jobs failed, final retry at 4 AM');
          await this.executePrewarm('4:00 AM UTC (retry 2)');
        }
      }

      // Also expose as callable method for one-time trigger after deploy
      async executePrewarm(jobName: string): Promise<void> { ... }
    }
    ```

    **executePrewarm logic:**
    1. Define 10 cuisines: ['italian', 'mexican', 'chinese', 'indian', 'thai', 'french', 'japanese', 'mediterranean', 'american', 'korean'].
    2. For each cuisine: call `spoonacularService.search('', { cuisines: [cuisine] }, 10)` sorted by popularity. Collect all recipe IDs. This costs ~10 search calls = ~10 points.
    3. Deduplicate collected recipe IDs.
    4. Call `spoonacularService.getRecipeInformationBulk(allIds)` for up to 100 recipes. Cost: 1 + (N-1)*0.5 points (~50 points for 100 recipes).
    5. Validate each recipe via `validateRecipe()`. Skip invalid ones, log warning with reason.
    6. Map valid recipes via `mapSpoonacularToRecipe()`.
    7. Store via `cacheService.upsertRecipes(mappedRecipes)`.
    8. Pre-warm 10 popular search queries: ['chicken', 'pasta', 'vegan', 'dessert', 'salad', 'soup', 'breakfast', 'quick dinner', 'healthy', 'keto']. For each, call `spoonacularService.search(query, {}, 20)`, validate, map, store recipes and cache search results.
    9. Record `lastSuccessfulRun = new Date()`.
    10. Log: `Pre-warm complete: ${validCount} recipes cached, ${skippedCount} skipped, ${searchQueries.length} searches cached`.

    **Total estimated quota cost for pre-warm:** ~10 (cuisine searches) + ~50 (bulk fetch 100) + ~10 (search query warm) = ~70 points. This exceeds the 50-point budget. **Optimization required**: Use complexSearch with `addRecipeInformation=true` and `addRecipeNutrition=true` to get full recipe info in one call (avoids separate bulk fetch). Or reduce cuisine count to 5, fetch 20 recipes per cuisine using complexSearch with addRecipeInformation=true. Cost: 5 * (1 + 0.01*20 + 0.025*20) = 5 * 1.7 = 8.5 points for recipes. Then 10 * ~1.7 = ~17 for search queries. Total: ~25 points, well within budget.

    **Revised approach**: Use complexSearch with `addRecipeInformation=true` and `addRecipeNutrition=true` to get full recipe data in one call. Reduces from 20+ API calls to ~15. Cost: ~25 points.
    - 5 cuisine batches of 20 recipes each = 100 recipes, 5 API calls.
    - 10 search queries of 20 results each = 10 API calls.
    - Total: ~15 API calls, ~25 points.

    **shouldRetry(minHours)**: returns true if lastSuccessfulRun is null or older than minHours.

    **Write tests** with mocked SpoonacularService, SpoonacularCacheService, PrismaService. Test all 10 behaviors. For @Cron tests, call methods directly (don't test cron scheduling itself).
