    Get API key from ConfigService at runtime (not hardcoded in module config). Use `HttpModule.registerAsync` or inject ConfigService in SpoonacularService to add apiKey to requests.

    **Step 8: Create SpoonacularService** (backend/src/spoonacular/spoonacular.service.ts)
    Implement:
    - `search(query, filters, number, offset)`: calls `/recipes/complexSearch` with addRecipeInformation=true, sort=popularity, number param. Returns SpoonacularRecipe[]. Point cost: 1 + 0.01*results + 0.025*results (with addRecipeInformation).
    - `getRecipeInformationBulk(ids: number[])`: calls `/recipes/informationBulk?ids=X,Y,Z&includeNutrition=true`. Point cost: 1 + (ids.length - 1) * 0.5.
    - `checkQuota(pointCost: number): Promise<boolean>`: reads ApiQuotaUsage for today, returns false if insufficient. Logs warning at 80%.
    - `incrementQuotaUsage(points: number)`: upserts ApiQuotaUsage for today.
    - `hasQuotaRemaining(): Promise<boolean>`: convenience check with 1-point buffer.
    - Circuit breaker: `consecutiveFailures` counter, `circuitOpenUntil` Date|null. Opens after 5 failures for 15 minutes.
    - Rate limiting: `lastRequestTime` tracker, enforces 1000ms between requests.
    - Logger: NestJS Logger, INFO for successful calls, WARN for quota threshold, ERROR for failures.
    - API key from ConfigService: `this.configService.get('SPOONACULAR_API_KEY')`, added as query param `apiKey`.
    - Daily quota limit from ConfigService: `this.configService.get('SPOONACULAR_DAILY_QUOTA') || 50`.
