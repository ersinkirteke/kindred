    2. Create `ScanUploadReducer.swift`:
       - `@Reducer public struct ScanUploadReducer`
       - State:
         - `var image: UIImage`
         - `var scanType: ScanType`
         - `var uploadProgress: Double = 0` — 0.0 to 1.0
         - `var uploadState: UploadState = .compressing`
         - `var scanJob: ScanJob? = nil`
         - `var error: String? = nil`
         - `var isOfflineQueued: Bool = false`
         - `enum UploadState: Equatable { case compressing, uploading, processing, completed, failed }`
       - Dependencies:
         - `@Dependency(\.apolloClient) var apolloClient`
         - `@Dependency(\.continuousClock) var clock`
       - Actions:
         - `startUpload` — begins compression → upload pipeline
         - `compressionCompleted(Data)` — JPEG data ready, begin upload
         - `compressionFailed` — set error state
         - `uploadProgressUpdated(Double)` — update progress indicator
         - `uploadCompleted(ScanJob)` — store job, transition to processing state
         - `uploadFailed(String)` — set error, show retry button
         - `retryUpload` — retry from compression
         - `cancelUpload` — cancel in-flight upload, dismiss
         - `backToPatryTapped` — delegate dismiss (processing continues)
         - `delegate(Delegate)` — `dismissed`, `uploadStarted(ScanJob)`
       - Upload pipeline implementation:
         a. Compress: `image.compressForUpload()` wrapped in Task on background thread. Use autoreleasepool.
         b. If offline (check connectivity): queue photo data in memory, set isOfflineQueued = true, return
         c. Upload: Use Apollo client to execute UploadScanPhoto mutation with compressed data as GraphQLFile
         d. On success: transition to .processing state, show "Analyzing your photo..." animation
         e. On failure: set uploadState = .failed with error message, show Retry button
       - Cancel support: Use `.cancellable(id: CancelID.upload)` on upload effect, cancel on .cancelUpload
       - Offline queue: Store compressed Data in memory (user decision: "Failed photo kept until app restart"). Monitor connectivity via NotificationCenter or PantryReducer's connectivity state. Auto-retry when connectivity returns.

    3. Create `ScanUploadView.swift`:
       - Full-screen view showing captured photo with upload overlay:
         - Photo as background (blurred/dimmed during upload)
         - Overlay states based on uploadState:
           a. `.compressing`: "Preparing photo..." with indeterminate ProgressView
           b. `.uploading`: Circular progress indicator (ProgressView(value: uploadProgress)) with percentage text. Cancel button below.
           c. `.processing`: "Analyzing your photo..." with animated scanning effect. Animated dots or pulse effect. "Back to Pantry" button visible. Reduce Motion: replace animation with static text + spinner.
           d. `.completed`: Checkmark animation (brief, auto-dismisses)
           e. `.failed`: Error message in red, "Retry" button (accent color), "Cancel" text button
         - All text white on dark overlay
         - Accessibility: progress announced, error announced, cancel/retry buttons labeled
       - If isOfflineQueued: show "Photo saved — will upload when back online" banner with cloud.fill icon
