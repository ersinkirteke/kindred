  <action>
    Create a TCA reducer managing the onboarding flow:

    ```swift
    @Reducer
    public struct OnboardingReducer {
        @ObservableState
        public struct State: Equatable {
            public var currentStep: Int = 0
            public var totalSteps: Int = 4

            // Sign-in step state
            public var isSigningIn = false
            public var signInError: String?
            public var isAuthenticated = false

            // Dietary preferences step state
            public var selectedDietaryPrefs: Set<String> = []

            // Location step state
            public var locationAuthStatus: LocationAuthStatus = .notDetermined
            public var selectedCity: String?
            public var showCityPicker = false

            // Voice teaser step state (no additional state needed)

            public init() {}
        }

        public enum LocationAuthStatus: Equatable {
            case notDetermined
            case authorized
            case denied
        }

        public enum Action: Equatable {
            // Navigation
            case nextStep
            case skipStep
            case completeOnboarding

            // Sign-in step
            case appleSignInTapped
            case googleSignInTapped
            case signInSucceeded
            case signInFailed(String)
            case continueAsGuestTapped

            // Dietary preferences step
            case toggleDietaryPref(String)

            // Location step
            case requestLocationPermission
            case locationAuthChanged(LocationAuthStatus)
            case showManualCityPicker
            case dismissCityPicker
            case citySelected(String)

            // Voice teaser step
            case tryVoiceNowTapped
            case setupVoiceLaterTapped
        }
    }
    ```

    Reducer body:
    - `nextStep`: Increment currentStep. If currentStep >= totalSteps, send .completeOnboarding.
    - `skipStep`: Same as nextStep (all steps are skippable).
    - `completeOnboarding`: No state change — parent handles persisting `@AppStorage("hasCompletedOnboarding")` and dismissing.
    - `appleSignInTapped`: Set isSigningIn=true, signInError=nil. Call signInClient.signInWithApple(). On success: send .signInSucceeded. On cancellation: isSigningIn=false (no error). On other error: send .signInFailed.
    - `googleSignInTapped`: Same pattern for Google.
    - `signInSucceeded`: Set isSigningIn=false, isAuthenticated=true. Auto-advance to next step after brief delay (0.5s).
    - `signInFailed`: Set isSigningIn=false, signInError to message.
    - `continueAsGuestTapped`: Send .nextStep (proceed without signing in).
    - `toggleDietaryPref`: Toggle string in selectedDietaryPrefs set. Save to UserDefaults ("dietaryPreferences") as JSON-encoded Set<String> — uses SAME key as Phase 6 dietary chip bar for consistency.
    - `requestLocationPermission`: Call locationClient.requestAuthorization(). Observe status change.
    - `locationAuthChanged`: Update locationAuthStatus. If .authorized, auto-advance after getting city name. If .denied, show manual fallback.
    - `showManualCityPicker`: Set showCityPicker=true.
    - `citySelected`: Store selectedCity, save to @AppStorage("selectedCity"), dismiss picker, advance.
    - `tryVoiceNowTapped`: Send .completeOnboarding first (mark onboarding as done before launching voice flow). Parent will trigger voice upload presentation.
    - `setupVoiceLaterTapped`: Send .completeOnboarding.

    Dependencies: `@Dependency(\.signInClient)`, `@Dependency(\.locationClient)` (from FeedFeature).

    IMPORTANT: Dietary preferences use the SAME UserDefaults key "dietaryPreferences" as Phase 6 to ensure consistency. Changes in onboarding appear immediately in the feed.
