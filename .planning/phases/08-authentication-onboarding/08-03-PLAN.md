    **GuestMigrationClient.swift:**
    Create a TCA @DependencyClient for guest-to-authenticated data migration:

    ```swift
    @DependencyClient
    public struct GuestMigrationClient: Sendable {
        /// Upload all guest data to backend and clean up local storage
        public var migrateGuestData: @Sendable () async throws -> Void
        /// Check if there's pending guest data to migrate
        public var hasPendingMigration: @Sendable () async -> Bool
    }
    ```

    Live implementation:
    1. Read guestUserId from UserDefaults
    2. Fetch all GuestBookmark and GuestSkip records from SwiftData via GuestSessionClient
    3. Read dietary preferences from UserDefaults ("dietaryPreferences")
    4. Execute GraphQL mutation `MigrateGuestData(guestUserId:, bookmarks:, skips:, dietaryPreferences:)` — if this mutation doesn't exist in the schema, create a placeholder that logs the data and succeeds (backend mutation will be added when needed)
    5. On success: delete local GuestBookmark and GuestSkip records via GuestSessionClient. Clear "guestUserId" from UserDefaults (or keep for analytics — user decision says link for analytics continuity, so KEEP the UUID but mark as migrated with a "guestMigrated" boolean flag)
    6. On failure: throw error (caller handles retry)

    hasPendingMigration: Check if any GuestBookmark or GuestSkip records exist locally.

    Register in DependencyValues as `\.guestMigrationClient`. Test value: unimplemented.

    **AppReducer.swift (major modification):**
    Add auth state management, auth gate presentation, onboarding state, and migration:

    New State fields:
    ```swift
    var currentAuthState: AuthState = .guest
    @Presents var authGate: SignInGateReducer.State?
    var showOnboarding: Bool = false  // Controlled by KindredApp based on @AppStorage
    var isMigrating: Bool = false
    var migrationRetryCount: Int = 0
    var pendingGatedAction: GatedAction? = nil  // The action to execute after sign-in
    ```

    New types:
    ```swift
    enum GatedAction: Equatable {
        case bookmark(recipeId: String, recipeName: String, imageUrl: String?, cuisineType: String?)
        case toggleBookmark(recipeId: String)
        case listenToRecipe(recipeId: String)
    }
    ```

    New Actions:
    ```swift
    case authStateChanged(AuthState)
    case authGate(PresentationAction<SignInGateReducer.Action>)
    case authGateRequested(GatedAction)  // From child reducers
    case onboarding(OnboardingReducer.Action)  // Forward from onboarding
    case startMigration
    case migrationSucceeded
    case migrationFailed
    case retryMigration
    case observeAuth
    ```

    Key reducer logic:

    1. **authGateRequested**: Check cooldown first: read `UserDefaults("lastGateDismissedAt")`. If within 5 minutes, silently drop the action (don't show gate). Otherwise, store pendingGatedAction and present authGate with `SignInGateReducer.State()`.

    2. **authGate(.presented(.signInSucceeded(user)))**: Set currentAuthState to .authenticated(user). Dismiss authGate (set to nil). Execute pendingGatedAction if present. Start migration via .startMigration.

    3. **authGate(.presented(.dismissed))** / **authGate(.presented(.continueAsGuestTapped))**: Dismiss authGate. Record `Date()` to UserDefaults("lastGateDismissedAt") for cooldown. Clear pendingGatedAction.

    4. **startMigration**: Set isMigrating=true. Call guestMigrationClient.migrateGuestData(). On success: send .migrationSucceeded. On failure: send .migrationFailed.

    5. **migrationFailed**: Increment migrationRetryCount. If <= 3, schedule retry with exponential backoff (2s, 4s, 8s). Else stop retrying (data stays local).

    6. **Execute pendingGatedAction**: Based on GatedAction type, forward the appropriate action to feed or voice playback reducers. For .bookmark: send .feed(.swipeCard(recipeId, .right)). For .toggleBookmark: send .feed(.recipeDetail(.presented(.toggleBookmark))). For .listenToRecipe: send .voicePlayback(.startPlayback(...)).

    7. **Forward auth gate requests from feed/voicePlayback**: Listen for new delegate actions from child reducers indicating a gated action was attempted.

    8. **observeAuth**: On app launch, call signInClient.observeAuthState() and forward changes to .authStateChanged.

    Add .ifLet composition for authGate:
    ```swift
    .ifLet(\.$authGate, action: \.authGate) {
        SignInGateReducer()
    }
    ```

    CRITICAL: The swipe bookmark in FeedReducer currently saves locally for guests. In the next task, we modify FeedReducer to check auth state first and emit a delegate action instead of saving directly when user is guest.
