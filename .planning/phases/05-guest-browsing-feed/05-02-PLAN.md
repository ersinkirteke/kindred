    **Move and completely rewrite** the existing `FeedReducer.swift` from `Sources/FeedReducer.swift` to `Sources/Feed/FeedReducer.swift`. Delete the old file. Also delete `Sources/FeedPlaceholder.swift` (no longer needed).

    Implement the full FeedReducer with TCA @Reducer macro:

    **State (@ObservableState):**
    - `cardStack: [RecipeCard]` — Current visible cards (from API, minus already-swiped)
    - `swipeHistory: [SwipedRecipe]` — Last 3 swiped cards for undo (max 3 per user decision)
    - `isLoading: Bool` — True during initial load
    - `isRefreshing: Bool` — True during pull-to-refresh
    - `location: String` — Current city name, default "Istanbul"
    - `isOffline: Bool` — From NetworkMonitorClient
    - `hasNewRecipes: Bool` — Silent refresh indicator (show "New recipes available" after reconnect)
    - `error: String?` — API error message
    - `currentPage: Int` — Pagination offset
    - `hasMorePages: Bool` — Whether more recipes available
    - `bookmarkCount: Int` — For Me tab badge
    - `@Presents var recipeDetail: RecipeDetailReducer.State?` — Navigation to detail (Plan 03 will define RecipeDetailReducer)
    - `path: NavigationPath` — For NavigationStack programmatic navigation

    **Actions:**
    - `onAppear` — Load initial recipes + start network monitoring
    - `recipesLoaded(Result<[RecipeCard], Error>)` — Handle API response
    - `swipeCard(String, SwipeDirection)` — Swipe a card left/right by recipe ID
    - `undoLastSwipe` — Restore last swiped card to top of stack
    - `loadMoreRecipes` — Pagination: fetch next batch when 3 cards remain
    - `moreRecipesLoaded(Result<[RecipeCard], Error>)` — Append to stack
    - `refreshFeed` — Pull-to-refresh with .fetchIgnoringCacheData
    - `refreshCompleted(Result<[RecipeCard], Error>)` — Handle refresh result
    - `changeLocation(String)` — Change city, reload feed
    - `connectivityChanged(Bool)` — Network state change
    - `silentRefreshCompleted(Result<[RecipeCard], Error>)` — Background refresh after reconnect
    - `acknowledgeNewRecipes` — User pulled to see new recipes
    - `openRecipeDetail(String)` — Navigate to detail screen
    - `recipeDetail(RecipeDetailReducer.Action)` — Child detail actions
    - `bookmarkCountLoaded(Int)` — Update badge count

    **Reducer body:**
    ```
    @Dependency(\.apolloClient) var apolloClient
    @Dependency(\.guestSession) var guestSession
    @Dependency(\.networkMonitor) var networkMonitor
    ```

    Key logic:
    - `.onAppear`: Fetch ViralRecipesQuery with .returnCacheDataAndFetch policy, start network monitoring stream, load bookmark count
    - `.swipeCard`: Remove card from stack, add to swipeHistory (cap at 3), persist via guestSession. If direction == .right, call bookmarkRecipe. If .left, call skipRecipe. If stack count <= 3 and hasMorePages, trigger .loadMoreRecipes. Fire HapticFeedback.medium() for swipe.
    - `.undoLastSwipe`: Pop from swipeHistory, insert at index 0 of cardStack, call undoSkip/unbookmarkRecipe. Fire HapticFeedback.light().
    - `.refreshFeed`: Use cachePolicy .fetchIgnoringCacheData, reset currentPage to 0
    - `.changeLocation`: Set location, clear cardStack, set isLoading=true, fetch with new location. Use .fetchIgnoringCacheData for location change (not cache).
    - `.connectivityChanged(true)`: If was offline, do silent background fetch. Set hasNewRecipes = true on success.
    - `.connectivityChanged(false)`: Set isOffline = true
    - `.loadMoreRecipes`: Fetch Recipes query with limit=10, offset=currentPage*10, append results

    Pre-fetch RecipeDetailQuery for current top card silently (per user decision): When cardStack is not empty, fire a prefetch effect for cardStack[0].id using RecipeDetailQuery with .returnCacheDataAndFetch. This warms the Apollo cache for instant detail loading.

    **IMPORTANT:** For the `recipeDetail` navigation state, use `@Presents` and a simple optional `RecipeDetailReducer.State?` pattern. Plan 03 creates RecipeDetailReducer — for now, just declare the state/action type references. If RecipeDetailReducer doesn't exist yet at build time, use a forward declaration or stub. The actual integration will happen in Plan 04.

    Individual card with DragGesture per research Pattern 1:
    - `CardSurface` container with NO inner padding (images edge-to-edge at top)
    - Top half: Kingfisher `KFImage` hero image with `.resizable().aspectRatio(contentMode: .fill)` clipped to card bounds. Progressive blur-up loading: `.placeholder { Rectangle().fill(Color.kindredDivider).blur(radius: 10) }` with `.fade(duration: 0.25)`. Add ViralBadge overlay when isViral.

      - Flame icon + calories (if available)
      - Heart icon + formatted loves count (e.g., "2.3k")
      - Use .kindredCaption() for metadata, .kindredTextSecondary color
    - DragGesture on the card:
      - `.onChanged`: Update offset (x = translation.width, y = translation.height * 0.4 dampened), rotation = translation.width / 10
      - `.onEnded`: If abs(translation.width) > 200pt threshold → animate off-screen (offset.width = +/-500) and call onSwipe(direction). Else snap back with `.spring(response: 0.3, dampingFraction: 0.6)`.
    - State: `@State private var offset: CGSize = .zero`, `@State private var rotation: Double = 0`
    - Callbacks: `let onSwipe: (SwipeDirection) -> Void`, `let onTap: () -> Void`

    - `.accessibilityAction(named: "Skip") { onSwipe(.left) }`
    - `.accessibilityAction(named: "View details") { onTap() }`
