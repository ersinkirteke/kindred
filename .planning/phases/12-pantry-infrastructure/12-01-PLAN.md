**3. Create PantryService (`backend/src/pantry/pantry.service.ts`):**

Inject `PrismaService`. Implement:

- `findAllForUser(userId: string, sinceTimestamp?: Date)`: Returns all non-deleted items for user. If sinceTimestamp provided, returns only items with `updatedAt > sinceTimestamp` (for sync). Order by `updatedAt` desc.

- `addItem(input: AddPantryItemInput)`:
  1. Call `normalizeIngredient(input.name)` to get canonical name
  2. Look up catalog entry for auto-filling `foodCategory` if not provided
  3. Check for existing non-deleted item with same `normalizedName` AND same `userId`
  4. If exists: merge quantities (parse as numbers if possible, otherwise concatenate with " + "). Update the existing item's `updatedAt`.
  5. If not exists: create new PantryItem with all fields

- `bulkAddItems(input: BulkAddPantryItemsInput)`: Loop through items, call `addItem` for each (reuses normalization + merge logic). Return array of created/updated items.

- `updateItem(id: string, userId: string, input: UpdatePantryItemInput)`: Find item by id AND userId (security check), update fields. If name changed, re-normalize.

- `deleteItem(id: string, userId: string)`: Soft delete — set `isDeleted: true`, update `updatedAt`.

- `searchCatalog(query: string, lang: string)`: Search IngredientCatalog where canonicalName, canonicalNameTR, or any alias contains the query (case-insensitive). Use `mode: 'insensitive'` on string comparisons. For aliases array, use `has` with lowercased query. Limit to 10 results. Support `lang` parameter: if "tr", sort Turkish matches first.

- `private normalizeIngredient(inputName: string): Promise<string>`: Lowercase + trim input. Search IngredientCatalog for match on canonicalName (insensitive), canonicalNameTR (insensitive), or aliases array (has lowercased value). Return `catalogEntry.canonicalName` if found, otherwise return the lowercased/trimmed input as-is. For unknown items, also auto-create a new IngredientCatalog entry with `canonicalName = lowercased input`, empty aliases, null category/shelfLife (accept and learn per user decision).
