  **B1: Search — Keyword Search (SEARCH-01, SEARCH-02)**

  - Tap a search result card — verify recipe detail opens correctly

  **B2: Search — Debounce Verification (SEARCH-03)**
  - Type "chi" quickly — verify NO network request fires (< 3 characters)
  - Type "chicken soup" — verify results appear only AFTER typing stops (300ms debounce)
  - The 10-query test list (run all sequentially):
    1. `pasta`
    2. `chicken soup`
    3. `vegan chocolate cake`
    4. `salmon`
    5. `tacos`
    6. `quinoa salad`
    7. `bread`
    8. `stir fry`
    9. `mushroom risotto`
    10. `breakfast burrito`

  **B3: Search — Cache Verification**
  - After running the 10-query test above, search `pasta` again
  - Wait 5 seconds, search `pasta` once more
  - Backend verification (SSH to production server):
    ```bash
    # Confirm log access pattern with user first — use PM2 or journalctl:
    pm2 logs kindred --lines 500 --nostream | grep -E "Searching recipes|stale cache|search failed"
    # OR:
    journalctl -u kindred-backend --since "15 minutes ago" | grep -E "Searching recipes|stale cache|search failed"
    ```
  - Expected: First `pasta` search shows 1 Spoonacular API call in logs. Second `pasta` search shows 0 calls (cache hit).
  - Count total Spoonacular calls across all 10 queries + 2 repeats — should be single-digit (most results cached from prior sessions or first run)

  **B4: Dietary Filter Pass-Through (FILTER-01, FILTER-02)**
  - Select "Vegan" dietary chip and search "cake" — verify returned recipes are genuinely vegan
  - Select "Gluten-Free" chip and search "bread" — verify results are gluten-free
  - Verify "Gluten-Free" maps to intolerances param (not diet param) — functional evidence: results should include non-vegan gluten-free recipes
  - Clear all chips — verify search returns unfiltered results

  **B5: AVSpeech Compact Fallback (VOICE-03)**
  - On fresh TestFlight install (without downloading enhanced voices in Settings > Accessibility > Spoken Content)
  - Play any recipe — verify narration plays using default/compact quality voice without error
  - Note: Enhanced voice is NOT expected on fresh install — compact fallback is the success criterion

  **B6: Return to Browse Mode**
  - After searching, tap Cancel or clear search
  - Verify popular feed restores with existing dietary chips and scroll position intact
  - Verify already-swiped cards are not re-shown
