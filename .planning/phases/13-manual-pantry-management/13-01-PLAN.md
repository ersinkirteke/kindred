**Create `AddEditItemReducer.swift`:**

```swift
@Reducer
public struct AddEditItemReducer {
    public enum Mode: Equatable {
        case add
        case edit(UUID)
    }

    @ObservableState
    public struct State: Equatable {
        public var mode: Mode
        public var userId: String
        public var name: String = ""
        public var quantity: String = "1"
        public var unit: String? = nil
        public var storageLocation: StorageLocation = .pantry
        public var foodCategory: FoodCategory? = nil
        public var expiryDate: Date? = nil
        public var notes: String = ""
        public var showNotesField: Bool = false
        public var isSubmitting: Bool = false
        public var suggestions: [(name: String, unit: String?, category: FoodCategory?)] = [] // Won't work in Equatable — see below
        public var duplicateWarning: String? = nil
        public var suggestedCategory: FoodCategory? = nil
        @Presents public var confirmDiscard: AlertState<Action.Alert>?
        @Presents public var confirmDelete: AlertState<Action.Alert>?

        // Track original values for isDirty detection
        var originalName: String = ""
        var originalQuantity: String = "1"
        // ... store all original values for isDirty

        public var isValid: Bool { !name.trimmingCharacters(in: .whitespaces).isEmpty }
        public var isDirty: Bool { /* compare current vs original values */ }
    }
}
```

**Important:** The `suggestions` array contains tuples which don't auto-conform to `Equatable`. Create a small `struct AutocompleteSuggestion: Equatable, Sendable` with `name`, `unit`, `category` fields instead of tuples.

**Actions:**
- `nameChanged(String)` — update name, debounce 300ms then fetch suggestions + check duplicate + query category auto-suggest via NetworkClient
- `quantityChanged(String)` — update quantity
- `unitChanged(String?)` — update unit
- `storageLocationChanged(StorageLocation)` — update storage location, re-check duplicate
- `foodCategoryChanged(FoodCategory?)` — update category
- `expiryDateChanged(Date?)` — update expiry date
- `showNotesFieldTapped` — set showNotesField = true
- `notesChanged(String)` — update notes
- `suggestionTapped(AutocompleteSuggestion)` — fill name, unit, category from suggestion
- `suggestionsLoaded([AutocompleteSuggestion])` — update suggestions
- `categorySuggested(FoodCategory)` — set suggestedCategory (only if user hasn't manually set one)
- `duplicateCheckResult(String?)` — set duplicateWarning text (e.g., "You already have Milk in Fridge")
- `submitTapped` — validate, create PantryItemInput, call pantryClient.addItem or updateItem
- `itemAdded(StorageLocation)` — batch mode: clear fields, retain storageLocation, haptic feedback
- `cancelTapped` — if isDirty, show confirmDiscard alert; otherwise send delegate(.cancelled)
- `deleteTapped` — show confirmDelete alert
- `alert(PresentationAction<Alert>)` — handle alert actions
- `delegate(Delegate)` — .itemSaved, .itemDeleted(UUID), .cancelled

**Reducer body logic:**
- For `nameChanged`: set `state.name = text`. If text count >= 2, return `.run` that:
  1. Debounces 300ms using `try await clock.sleep(for: .milliseconds(300))` with cancellation ID
  2. Fetches suggestions via `pantryClient.fetchSuggestions(userId, text)`
  3. Checks duplicate via `pantryClient.checkDuplicate(userId, text, storageLocation)`
  4. Queries category via `apolloClient.searchIngredients(query: text)` (wrap in try? to not block on network failure)
  5. Sends `.suggestionsLoaded`, `.duplicateCheckResult`, `.categorySuggested` actions
- For `submitTapped`: Build `PantryItemInput` from state. If mode == .add, call `pantryClient.addItem(input)`. If mode == .edit(id), call `pantryClient.updateItem(id, input)`. On success: if add mode, send `.itemAdded(state.storageLocation)`; if edit mode, send `.delegate(.itemSaved)`.
- For `itemAdded`: Clear name, quantity="1", unit=nil, foodCategory=nil, expiryDate=nil, notes="", showNotesField=false, suggestions=[], duplicateWarning=nil. Keep storageLocation. Fire haptic (UINotificationFeedbackGenerator success).
- For `deleteTapped`: Show confirmDelete AlertState with "Delete {name}?" message and destructive confirm button.
- For confirm delete alert action: Call `pantryClient.deleteItem(id)` then send `.delegate(.itemDeleted(id))`.
- For `cancelTapped`: If isDirty, show confirmDiscard AlertState; else send `.delegate(.cancelled)`.
- For confirm discard alert action: Send `.delegate(.cancelled)`.

**Dependencies:**
