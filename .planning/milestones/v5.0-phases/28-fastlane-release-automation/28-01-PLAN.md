<objective>
Add a `precheck` lane to Kindred/fastlane/Fastfile that validates workspace configuration before any upload action, then wire it into the existing `beta_internal` and `release` lanes as a fail-fast gate.

Purpose: A single typo in Release.xcconfig or a missing .p8 file currently surfaces 10–30 minutes into a build, after the compiler has done all its work. Precheck catches configuration errors in under 10 seconds, before the first build step runs, saving the solo dev from costly rework on first real submission.

Output: Fastfile with a new `precheck` lane plus two one-line additions (`precheck` called as first statement) inside `beta_internal` and `release`.
</objective>

  <name>Task 1: Add precheck lane with 6 fail-fast checks</name>
  <files>Kindred/fastlane/Fastfile</files>
  <action>
    Append a new `precheck` lane inside the `platform :ios do` block, after `before_all` and before `beta_internal`. The lane must perform these six checks in order and halt with `UI.user_error!` on the first failure. Use `UI.success` after each passing check so the solo dev can see progress when running locally.

    Check 1 — Release.xcconfig placeholders:
    - Read `../Config/Release.xcconfig` relative to the fastlane dir
    - Fail with message "❌ Release.xcconfig contains REPLACE_WITH_ placeholder values — fix before release" if the content includes the literal string `REPLACE_WITH_`
    - On pass: `UI.success("✅ Release.xcconfig: no REPLACE_WITH_ placeholders")`

    Check 2 — .env file completeness:
    - Verify `.env` exists in the current working directory (the fastlane dir)
    - Fail with "❌ fastlane/.env file not found — copy from .env.default and populate values" if missing
    - Read the file, iterate over required keys `["APP_STORE_CONNECT_API_KEY_ID", "APP_STORE_CONNECT_ISSUER_ID", "APP_STORE_CONNECT_API_KEY_FILEPATH"]`
    - For each missing key, fail with "❌ fastlane/.env missing required key: #{key}"
    - On pass: `UI.success("✅ .env: all required App Store Connect API keys present")`

    Check 3 — .p8 API key file readable:
    - Read `ENV["APP_STORE_CONNECT_API_KEY_FILEPATH"]` (loaded automatically from .env)
    - Fail with "❌ .p8 API key file not readable at #{p8_path} — regenerate from App Store Connect → Users & Access → Integrations → API Keys" if `File.exist?` is false OR `File.readable?` is false
    - On pass: `UI.success("✅ .p8 API key: readable at #{p8_path}")`

    Check 4 — Metadata files exist and non-empty in both locales:
    - Iterate over `%w[en-US tr]`
    - For each locale, iterate over `%w[name.txt subtitle.txt description.txt keywords.txt promotional_text.txt release_notes.txt support_url.txt privacy_url.txt marketing_url.txt]`
    - Path: `File.join(Dir.pwd, "metadata", locale, file)`
    - Fail with "❌ Missing or empty metadata file: metadata/#{locale}/#{file}" if `File.exist?` false OR `File.size` is zero
    - On pass (after all files checked): `UI.success("✅ Metadata: all 9 required files present and non-empty in en-US and tr")`
    - NOTE: tr/support_url.txt, tr/privacy_url.txt, tr/marketing_url.txt do not exist YET — they will be created by Plan 28-02 in the same wave. Precheck MUST still list them. It is only executed at beta/release time, after Plan 28-02 has landed. Do not add conditional logic to skip them.

    Check 5 — Screenshots (exactly 5 per locale):
    - Iterate over `%w[en-US tr]`
    - `Dir.glob(File.join(Dir.pwd, "screenshots", locale, "*.png"))`
    - Fail with "❌ Expected 5 screenshots in screenshots/#{locale}/, found #{count}" if count != 5
    - On pass: `UI.success("✅ Screenshots: 5 PNGs present in en-US and tr")`

    Check 6 — Sanity recap:
    - `UI.success("🎉 All pre-flight checks passed — safe to proceed with build")`

    Do NOT call `ensure_git_status_clean` inside precheck — `before_all` already enforces it, and precheck should focus on config/file checks that `before_all` does not cover.

    Use `desc "Pre-flight checks before any upload (fail fast on config errors)"` above the `lane :precheck do` declaration so `fastlane lanes` lists it cleanly.

    Preserve the existing `beta_internal`, `beta_external`, and `release` lane bodies byte-for-byte in this task — Task 2 wires precheck into them. This task must only ADD the precheck lane.
