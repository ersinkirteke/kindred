<objective>
Upload a fresh TestFlight build from current main via `fastlane beta_internal`, then run a 48–72 hour internal bake with 3+ testers against the Phase 22 pre-submission checklist and core flow guide. Produce a go/no-go report that gates Plan 28-05 (the actual App Store submission).

Purpose: This is STORE-04 — the TestFlight validation requirement. CONTEXT.md explicitly forbids reusing stale TestFlight builds: "Phase 28 starts by running `fastlane beta_internal` to upload a fresh build from current main (includes Phase 27.1 AdMob fixes + any post-Phase-22 changes)." The bake is the last chance to catch a regression before Apple sees the binary. If it fails, Phase 28 pauses and loops back into fix mode.

Output: A processed TestFlight build in ASC, a populated 28-04-BETA-REPORT.md with go/no-go decision, and the green light for Plan 28-05 to proceed with `fastlane release`.

**This plan is NOT autonomous.** The beta bake is a human-time activity (48–72 hours of wall clock time, real device testing by 3+ people). Task 1 is fully automated (Claude runs `fastlane beta_internal`). Task 2 is a blocking human-verify checkpoint that spans the bake window and ends with an explicit GO/NO-GO decision recorded in 28-04-BETA-REPORT.md.
</objective>
