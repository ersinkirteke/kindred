    **Create SpoonacularCacheService** (backend/src/spoonacular/spoonacular-cache.service.ts):

    ```typescript
    @Injectable()
    export class SpoonacularCacheService {
      private readonly TTL_MS = 6 * 60 * 60 * 1000; // 6 hours

      constructor(private prisma: PrismaService, private logger: Logger) {}

      // Check cache first, return recipes + staleness status
      async getCachedSearch(normalizedKey: string): Promise<{ recipes: Recipe[]; isStale: boolean } | null>;

      // Store search results: upsert SearchCache entry, upsert recipes by spoonacularId
      async cacheSearchResults(normalizedKey: string, recipes: MappedRecipeData[]): Promise<void>;

      // Get cached recipes by internal IDs
      async getCachedRecipes(recipeIds: string[]): Promise<Recipe[]>;

      // Normalize cache key: lowercase query, sort filter keys alphabetically, JSON.stringify
      normalizeCacheKey(query: string, filters: SearchFilters): string;

      // Check if a cache entry is stale
      isStale(cachedAt: Date): boolean;

      // Upsert recipes by spoonacularId (create if new, skip if exists)
      async upsertRecipes(recipes: MappedRecipeData[]): Promise<string[]>;

      // Remove stale cache entries (>24 hours)
      async cleanExpiredCache(): Promise<number>;
    }
    ```

    Key details:
    - `normalizeCacheKey`: `JSON.stringify({ q: query.trim().toLowerCase(), f: Object.keys(filters).sort().reduce(...) })`.
    - `getCachedSearch`: find SearchCache by normalizedKey, if found get recipes by IDs, return with `isStale` flag.
    - `upsertRecipes`: use `prisma.recipe.upsert` with `where: { spoonacularId }` — create if new, update if existing (refresh data). Create nested ingredients and steps using Prisma nested writes. Use a transaction to delete old ingredients/steps before creating new ones (avoids duplicates).
    - `cacheSearchResults`: upsert SearchCache entry (delete+create or upsert normalizedKey), then call upsertRecipes.
    - Include relations (ingredients, steps) when fetching cached recipes.
    - All dates use UTC.

    - Inject SpoonacularService, SpoonacularCacheService, PrismaService, Logger.
    - `searchRecipes(input: SearchRecipesInput): Promise<RecipeConnection>`:
      1. Normalize cache key from input (query + filters).
      2. Check cache via cacheService.getCachedSearch(normalizedKey).
      3. If cache HIT and fresh: build RecipeConnection from cached recipes.
      4. If cache HIT and stale: return cached recipes immediately, fire background refresh (non-awaited promise with .catch logging).
      5. If cache MISS: check quota via spoonacularService.hasQuotaRemaining().
         - If quota OK: call spoonacularService.search(), validate and map recipes via recipe-mapper, cache via cacheService, return.
