### Critical: SubscriptionClient Auth Token

**Root cause:** Phase 19 added `ClerkAuthGuard` to `SubscriptionResolver`, but iOS `SubscriptionClient.syncSubscriptionToBackend` (from Phase 9) was not updated to include the Authorization header.

**Impact:** All iOS StoreKit purchase verifications fail with 401 Unauthorized. Users appear as free tier despite successful purchase.

**Fix required in Phase 21:**
```swift
// SubscriptionClient.swift:152-155
let clerkToken = try await Clerk.shared.session?.getToken()
request.setValue("Bearer \(clerkToken)", forHTTPHeaderField: "Authorization")
```
