    **Colors.swift** — SwiftUI Color extensions reading from Asset Catalog:
    ```swift
    import SwiftUI

    public extension Color {
        static let kindredPrimary = Color("Primary", bundle: .module)
        static let kindredBackground = Color("Background", bundle: .module)
        static let kindredCardSurface = Color("CardSurface", bundle: .module)
        static let kindredAccent = Color("Accent", bundle: .module)
        static let kindredAccentDecorative = Color("AccentDecorative", bundle: .module)
        static let kindredTextPrimary = Color("TextPrimary", bundle: .module)
        static let kindredTextSecondary = Color("TextSecondary", bundle: .module)
        static let kindredDivider = Color("Divider", bundle: .module)
        static let kindredError = Color("Error", bundle: .module)
        static let kindredSuccess = Color("Success", bundle: .module)
    }
    ```

    NOTE: Since DesignSystem is an SPM package, color assets should be in the package's Resources directory (not the app's Assets.xcassets). Update Package.swift to include resources: `resources: [.process("Resources")]`. Create `Packages/DesignSystem/Sources/Resources/Colors.xcassets/` with the color sets. Use `bundle: .module` in Color initializers.

    **Typography.swift** — SF Pro font styles per locked decision (medium headings, light body):
    ```swift
    public extension Font {
        static func kindredLargeTitle() -> Font { .system(size: 34, weight: .medium, design: .default) }
        static func kindredHeading1() -> Font { .system(size: 28, weight: .medium, design: .default) }
        static func kindredHeading2() -> Font { .system(size: 22, weight: .medium, design: .default) }
        static func kindredHeading3() -> Font { .system(size: 18, weight: .medium, design: .default) }
        static func kindredBody() -> Font { .system(size: 18, weight: .light, design: .default) }  // WCAG AAA min 18sp
        static func kindredBodyBold() -> Font { .system(size: 18, weight: .medium, design: .default) }
        static func kindredCaption() -> Font { .system(size: 14, weight: .light, design: .default) }
        static func kindredSmall() -> Font { .system(size: 12, weight: .light, design: .default) }
    }
    ```

    All font methods return `Font` (not fixed sizes) so Dynamic Type scaling works automatically with SwiftUI's `.dynamicTypeSize()` modifier support.

  <action>
    Create 5 reusable components in DesignSystem/Sources/Components/:

    **1. KindredButton** — Primary CTA button with 56dp minimum touch target (WCAG AAA per ACCS-01):
    - Styles: `.primary` (terracotta background, white text), `.secondary` (outlined, terracotta border), `.text` (text only)
    - Minimum frame height: 56pt (matches 56dp touch target requirement)
    - Minimum frame width: 56pt for icon-only buttons, stretches for text
    - Corner radius: 12pt
    - Uses `.kindredAccent` background for primary, `.kindredTextPrimary` for text
    - Supports loading state (ProgressView replaces label)
    - Supports disabled state (opacity reduction)
    - Usage: `KindredButton("Listen", style: .primary) { action() }`

    **2. CardSurface** — Container view for recipe cards and content blocks:
    - Background: `.kindredCardSurface`
    - Corner radius: 16pt (per locked decision: rounded corners with padding)
    - Padding: 16pt inner padding
    - Optional shadow (light mode only — subtle, no shadow in dark mode)
    - Usage: `CardSurface { content }`

    **3. SkeletonShimmer** — Shimmer animation modifier for loading states:
    - ViewModifier applying linear gradient sweep animation
    - Pairs with SwiftUI `.redacted(reason: .placeholder)` (per research Pattern 4)
    - Animation: linear gradient sweep left-to-right, 1.5s duration, repeating
    - Gradient colors: clear → white at 30% opacity → clear
    - Usage: `RecipeCardView(recipe: .placeholder).redacted(reason: .placeholder).modifier(SkeletonShimmer())`
    - Also provide convenience: `.shimmer()` View extension

    **4. ErrorStateView** — Warm, friendly error display:
    - Title (heading2, medium weight)
    - Message body (body, light weight) — warm tone per locked decision: "Hmm, we can't find recipes right now. Check your connection and try again."
    - Retry button (KindredButton, primary style)
    - SF Symbol icon (e.g., "wifi.slash" for network, "exclamationmark.triangle" for generic)
    - Uses ContentUnavailableView (iOS 17+) as base per research example
    - Usage: `ErrorStateView(title: "Connection Issue", message: "...", icon: "wifi.slash") { retryAction() }`

    **5. EmptyStateView** — Friendly empty state display:
    - Similar to ErrorStateView but no retry button
    - SF Symbol icon
    - Title and subtitle text
    - Uses ContentUnavailableView
    - Usage: `EmptyStateView(title: "No Recipes Found", message: "Try changing your location.", icon: "fork.knife")`

    **All components must:**
    - Use DesignSystem Colors and Typography (no hardcoded hex or font values)
    - Support Dynamic Type scaling
    - Include SwiftUI Preview providers for visual verification
    - Be marked `public` for cross-package access
