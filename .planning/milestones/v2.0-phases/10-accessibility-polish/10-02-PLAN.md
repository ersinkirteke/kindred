**AppReducer.swift:**

1. Add `var isOffline: Bool = false` to `State`.

2. Add new actions:
   - `case startConnectivityMonitor` — starts the network monitor stream
   - `case connectivityChanged(Bool)` — updates isOffline, announces via VoiceOver, triggers auto-refresh

3. Add `@Dependency(\.networkMonitorClient) var networkMonitor` — reuse existing FeedFeature NetworkMonitorClient (already a TCA dependency).

4. In the reducer body, handle:
   - `.startConnectivityMonitor`: Start `networkMonitor.startMonitoring()` async stream, send `.connectivityChanged` for each value.
   - `.connectivityChanged(let isOffline)`:
     - Set `state.isOffline = isOffline`
     - VoiceOver announcement: `UIAccessibility.post(notification: .announcement, argument: isOffline ? "You're offline" : "Back online")` — per locked decision
     - When going back online (`!isOffline`): send `.feed(.fetchRecipes)` to auto-refresh feed per locked decision "Auto-refresh all screens when connectivity returns"
     - Return `.none` otherwise

5. In the `.onAppear`-equivalent (add to `.observeAuth` or create new `.appDidAppear` action): trigger `.startConnectivityMonitor`.

6. Forward `isOffline` to child states. Since FeedReducer already has its own isOffline from its own network monitor, the cleanest approach is to add `isOffline` as app-level state and have RootView pass it. OR send `.feed(.connectivityChanged(isOffline))` so FeedReducer gets it from AppReducer instead of its own monitor. Use the simpler approach: just add isOffline to AppReducer.State and use it in RootView for the offline banner. Child features that already monitor connectivity can keep their own — AppReducer's state is for the shared UI (banner, toast).

**RootView.swift:**

1. Import DesignSystem.
2. Add `OfflineBanner()` at the top of the TabView content when `store.isOffline == true`:
   ```swift
   VStack(spacing: 0) {
       if store.isOffline {
           OfflineBanner()
       }
       TabView(selection: ...) { ... }
   }
   ```
3. Send `.startConnectivityMonitor` in the existing `.onAppear` block (alongside `.observeAuth`).

Note: `import UIKit` is needed in AppReducer for `UIAccessibility.post`. Import it.

2. **Add MXMetricManagerSubscriber conformance:**
   - In `didFinishLaunchingWithOptions`: `MXMetricManager.shared.add(self)`
   - In `applicationWillTerminate`: `MXMetricManager.shared.remove(self)`
   - Implement `didReceive(_ payloads: [MXMetricPayload])`:
     - Log launch metrics: extract `applicationLaunchMetrics?.histogrammedTimeToFirstDraw` average, log via `Logger.performance`
     - Log hang metrics: count and total hang time
     - Log exit metrics: background and foreground abnormal exits
     - Warn if launch time exceeds 1s target
   - Implement `didReceive(_ payloads: [MXDiagnosticPayload])`:
     - Log crash and hang diagnostics via `Logger.performance`

   - Task identifier: `"com.ersinkirteke.kindred.recipe-refresh"`
   - In `didFinishLaunchingWithOptions`: register handler via `BGTaskScheduler.shared.register(forTaskWithIdentifier:using:)`
   - Handler: `handleRecipeRefresh(task:)` method:
     - Schedule next refresh
     - Set expiration handler
     - Fetch latest recipes (placeholder async call — actual implementation depends on Apollo client availability)
     - `task.setTaskCompleted(success:)`
   - `scheduleRecipeRefresh()` method:
     - `BGAppRefreshTaskRequest(identifier:)` with `earliestBeginDate = Date(timeIntervalSinceNow: 15 * 60)` (15 minutes per Claude's discretion)
     - Submit to `BGTaskScheduler.shared`
