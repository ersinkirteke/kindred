Per locked decision: "Use String(localized:) initializer for all user-facing strings. Xcode auto-extracts into String Catalog."

Systematic approach -- go through every view file and replace hardcoded strings:

**String replacement rules:**
1. `Text("Listen")` -> `Text(String(localized: "Listen"))`
2. `Button("Skip")` -> `Button(String(localized: "Skip"))`
3. `.accessibilityLabel("...")` -> `.accessibilityLabel(String(localized: "accessibility.view_name.element"))`
4. `.accessibilityHint("...")` -> `.accessibilityHint(String(localized: "accessibility.view_name.hint"))`
5. String interpolation: `"Photo of \(recipe.name)"` -> `String(localized: "Photo of \(recipe.name)")`
6. Tab labels: `Label("Feed", ...)` -> `Label(String(localized: "Feed"), ...)`

**Key convention for accessibility strings:** Use dotted prefix: `accessibility.{view}.{element}` (e.g., `accessibility.recipe_card.skip`, `accessibility.mini_player.play`).

**Key convention for UI strings:** Use natural English text as the key for simple strings (e.g., `"Listen"`, `"Skip"`, `"Bookmark"`). Use dotted keys for context-specific strings (e.g., `"feed.end_of_stack.title"`).

**Files to modify (14 files across 3 groups):**

1. **Feed views** (9 files: RecipeCardView, SwipeCardStack, FeedView, DietaryChipBar, DietaryChip, ViralBadge, ForYouBadge, EndOfStackCard, CardCountIndicator):
   - All button labels, badge text, chip text, accessibility labels/hints
   - RecipeCardView: Listen, Skip, Bookmark button labels; metadata labels (minutes, calories); accessibility combined label; image alt text
   - ViralBadge/ForYouBadge: Badge text ("Viral", "For You")
   - EndOfStackCard: End of stack message text
   - DietaryChip/DietaryChipBar: Filter chip labels, accessibility hints
   - CardCountIndicator: Count display text

2. **RecipeDetail views** (4 files: RecipeDetailView, IngredientChecklistView, StepTimelineView, ParallaxHeader):
   - Section headers ("Ingredients", "Steps"), button labels, metadata labels
   - Accessibility strings for reading order elements
   - Offline message: "Voice narration requires internet", "Available when online"

3. **Location views** (1 file: LocationPickerView):
   - Search placeholder "Search cities...", "Use my location" text, headers
   - Accessibility label and hint for location pill

**Important:** Do NOT localize:
- Developer-facing strings (Logger messages, debug strings)
- UserDefaults keys
- GraphQL query strings
- Bundle identifiers or technical constants
