    **PlaybackState.swift:**
    - `PlaybackStatus` enum: `.idle`, `.loading`, `.buffering`, `.playing`, `.paused`, `.stopped`, `.error(String)`
    - `PlaybackSpeed` enum with Float rawValue: `.half(0.5)`, `.threeQuarter(0.75)`, `.normal(1.0)`, `.oneAndQuarter(1.25)`, `.oneAndHalf(1.5)`, `.double(2.0)` — with `next` computed property to cycle through speeds

    **AudioPlayerManager.swift:**
    - Actor-based class wrapping AVPlayer (not AVAudioPlayer — streaming requirement)
    - Properties: private player (AVPlayer?), private timeObserverToken (Any?), private statusObservation (NSKeyValueObservation?)
    - Methods:
      - `play(url: URL)` — creates AVPlayerItem, replaces current item, sets audioTimePitchAlgorithm to .timePitch, calls player.play()
      - `pause()` — player?.pause()
      - `resume()` — player?.play(), then set rate to current speed
      - `seek(to seconds: TimeInterval)` — uses CMTime with toleranceBefore/After of 0.5 seconds for smooth seeking
      - `setRate(_ rate: Float)` — player?.play() first then set rate (rate MUST be set AFTER play() per research pitfall)
      - `currentTimeStream()` -> AsyncStream<TimeInterval> — uses addPeriodicTimeObserver with 0.5s interval, stores token, removes in onTermination
      - `statusStream()` -> AsyncStream<PlaybackStatus> — observes player.timeControlStatus via KVO, maps to PlaybackStatus enum
      - `durationStream()` -> AsyncStream<TimeInterval> — observes currentItem?.duration
      - `cleanup()` — removes time observer token, removes KVO observation, pauses player, sets player to nil
    - CRITICAL: Store timeObserverToken and remove in cleanup() to prevent memory leaks (Research Pitfall 1)
    - Use `[weak self]` in all closures to prevent retain cycles
    - Add `.AVPlayerItemDidPlayToEndTime` notification observer for end-of-playback detection, yield `.stopped` in statusStream

    **AudioPlayerClient.swift:**
    - TCA dependency struct with closures:
      - `play: @Sendable (URL) async throws -> Void`
      - `pause: @Sendable () async -> Void`
      - `resume: @Sendable () async -> Void`
      - `seek: @Sendable (TimeInterval) async -> Void`

    Create the audio caching infrastructure and step synchronization engine:

    **CacheEntry.swift:**
    - `CacheEntry` struct (Codable, Equatable, Sendable): voiceId (String), recipeId (String), fileName (String), sizeBytes (Int64), lastAccessTime (Date), createdAt (Date)
    - Computed property `cacheKey: String` returning "\(voiceId)_\(recipeId)" for unique identification

    **LRUCache.swift:**
    - `VoiceCache` class managing audio files in FileManager.SearchPathDirectory.cachesDirectory (NOT .documentDirectory — per research anti-pattern)
    - Properties:
      - `maxCacheSizeBytes: Int64` = 500 * 1024 * 1024 (500MB default per user decision)
      - `cacheDirectory: URL` = cachesDirectory/VoiceNarrations/
      - `metadataKey: String` = "voiceCache_metadata" for UserDefaults storage
    - Methods:
      - `cacheAudio(voiceId: String, recipeId: String, data: Data) async throws -> URL` — saves file as "\(voiceId)_\(recipeId).m4a", creates CacheEntry, calls evictIfNeeded()
      - `getCachedAudio(voiceId: String, recipeId: String) async -> URL?` — returns file URL if exists, updates lastAccessTime (LRU touch)
      - `isCached(voiceId: String, recipeId: String) -> Bool` — quick check without updating access time
      - `evictIfNeeded() async throws` — sorts entries by lastAccessTime ascending, removes oldest until totalSize <= maxCacheSizeBytes. Delete file + remove metadata entry.
      - `getTotalCacheSize() -> Int64` — sum of all entry sizeBytes
      - `clearAll() throws` — removes entire cacheDirectory and UserDefaults metadata
    - Metadata persistence: encode/decode [CacheEntry] to/from UserDefaults via JSONEncoder/JSONDecoder
    - File naming: "\(voiceId)_\(recipeId).m4a" — caches audio per voice per recipe (user decision: multiple voices cached for same recipe)
    - Create cacheDirectory with withIntermediateDirectories: true

    **VoiceCacheClient.swift:**
    - TCA dependency struct with closures:
      - `cacheAudio: @Sendable (String, String, Data) async throws -> URL` (voiceId, recipeId, data)
      - `getCachedAudio: @Sendable (String, String) async -> URL?` (voiceId, recipeId)
      - `isCached: @Sendable (String, String) -> Bool` (voiceId, recipeId)
      - `totalCacheSize: @Sendable () -> Int64`
      - `clearCache: @Sendable () async throws -> Void`
    - liveValue: delegates to VoiceCache instance
    - testValue: returns nil for cache lookups, 0 for size
    - Register in DependencyValues as `voiceCacheClient`

    **StepSyncEngine.swift:**
    - `StepSyncEngine` struct (public, Sendable):
      - `static func currentStepIndex(at time: TimeInterval, timestamps: [TimeInterval]) -> Int?`
      - Uses BINARY SEARCH on sorted timestamps (NOT linear search — O(log n) per research Don't Hand-Roll)
      - Returns the index of the last timestamp that is <= current time
      - Returns nil if timestamps array is empty or time is before first timestamp
      - Edge case: if time >= last timestamp, returns last step index
    - Keep this as a pure function (no state) — reducer calls it with current time and timestamps from NarrationMetadata
