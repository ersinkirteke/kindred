  <action>
    Create the voice profile upload flow that lets users create voice profiles from audio clips:

    **VoiceUploadReducer.swift:**
    - @Reducer with @ObservableState State:
      - `selectedFileURL: URL?` — selected audio file
      - `fileName: String?` — display name of selected file
      - `fileDuration: TimeInterval?` — duration of selected audio (must be 30-60 seconds)
      - `voiceName: String = ""` — user-entered name for the voice profile
      - `isUploading: Bool = false`
      - `uploadProgress: Double = 0`
      - `error: String?`
      - `isValid: Bool` — computed: file selected, duration 30-60s, name not empty
      - `uploadComplete: Bool = false`
    - Actions:
      - `selectFile` — present file picker (UTType.audio)
      - `fileSelected(URL, TimeInterval)` — file chosen with duration
      - `voiceNameChanged(String)` — name text field changed
      - `upload` — start upload process
      - `uploadProgress(Double)` — progress update
      - `uploadCompleted(VoiceProfile)` — success
      - `uploadFailed(String)` — failure
      - `dismiss` — close the view
    - Dependencies:
      - `@Dependency(\.apolloClient) var apolloClient`
    - Upload flow:
      1. User selects audio file via .fileImporter (UTType.audio: .mp3, .m4a, .wav, .aac)
      2. Validate duration is 30-60 seconds using AVAsset.load(.duration)
      3. If invalid duration: show error "Voice clip must be 30-60 seconds"
      4. User enters voice profile name
      5. On upload: read file data, call backend REST endpoint (POST /api/voice-profiles/upload with multipart form — the backend uses REST for file uploads per v1.5 pattern, not GraphQL)
      6. Backend processes with ElevenLabs voice cloning (async — returns immediately with profile ID)
      7. Show success state and dismiss

    **VoiceUploadView.swift:**
    - Takes `@Bindable var store: StoreOf<VoiceUploadReducer>`
    - Layout (VStack):
      1. Header: "Create Voice Profile" (.kindredHeading2()), "Upload a 30-60 second voice clip" subtitle (.kindredBody(), .kindredTextSecondary)
      2. File selection area: dashed border rectangle with microphone.fill icon and "Select Audio File" button. Shows file name and duration when selected.
      3. Duration validation: green checkmark if 30-60s, red warning if outside range
      4. Name input: TextField "Voice Profile Name" with .kindredBody() font, styled border
      5. Upload button: KindredButton primary style, disabled until isValid. Shows ProgressView when uploading.
      6. Error display: if error present, show in .kindredError color
      7. Success state: checkmark.circle.fill icon, "Voice Profile Created!" message, "Done" button to dismiss
    - Use `.fileImporter(isPresented:allowedContentTypes:)` with [.audio] for file selection
    - All text 18sp+ (.kindredBody() minimum — ACCS-02)
    - Accessibility:
      - File select: `.accessibilityLabel("Select audio file for voice profile")`
      - Name field: `.accessibilityLabel("Voice profile name")`
      - Upload button: `.accessibilityLabel("Upload voice clip")`, `.accessibilityHint("Creates a voice profile from your audio clip")`
      - Duration indicator: `.accessibilityLabel("Duration: [X] seconds, [valid/invalid]")`
    - Navigation: This view is presented as a sheet from VoicePickerView's empty state "Create Voice Profile" button
    - DesignSystem styling: CardSurface for file area, .kindredAccent for buttons, .kindredBackground
