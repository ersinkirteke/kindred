    ```swift
    public extension String {
        var dietaryTagColor: Color {
            switch self.lowercased() {
            case "vegan": return .green
            case "vegetarian": return .green.opacity(0.8)
            case "keto": return .blue
            case "halal": return .purple
            case "gluten-free": return .orange
            case "dairy-free": return .cyan
            default: return .kindredTextSecondary
            }
        }
    }
    ```

    **2. Create RecipeDetailReducer.swift** at `Sources/RecipeDetail/RecipeDetailReducer.swift`:

    TCA @Reducer:

    **State (@ObservableState):**
    - `recipeId: String` — The recipe to display
    - `recipe: RecipeDetail?` — Loaded recipe data (nil while loading)
    - `isLoading: Bool = true`
    - `isBookmarked: Bool = false`
    - `checkedIngredients: Set<String>` — Set of ingredient IDs checked by user (session-only per user decision, not persisted for guests)
    - `error: String?`

    **Actions:**
    - `onAppear` — Load recipe detail, check bookmark status
    - `recipeLoaded(Result<RecipeDetail, Error>)` — Handle API response
    - `toggleBookmark` — Toggle bookmark status + persist via GuestSessionClient
    - `bookmarkStatusLoaded(Bool)` — Initial bookmark check result
    - `toggleIngredient(String)` — Toggle ingredient checked state
    - `listenTapped` — Placeholder for Phase 7 (disabled button, no-op)

    **Reducer body:**
    ```
    @Dependency(\.apolloClient) var apolloClient
    @Dependency(\.guestSession) var guestSession
    ```

    Key logic:
    - `.onAppear`: Fetch RecipeDetailQuery by recipeId. Thanks to Plan 02's pre-fetch, this should hit Apollo cache instantly (returnCacheDataAndFetch). Also check isBookmarked status.
    - `.toggleBookmark`: If isBookmarked, call unbookmarkRecipe. Else call bookmarkRecipe(recipeId, recipeName, imageUrl). Toggle isBookmarked. Soft bookmark limit: if bookmark count > 10, set a flag to show nudge (no hard block per user decision).
    - `.toggleIngredient`: Toggle ingredient ID in checkedIngredients set. Not persisted for guests (per user decision).

    Init: `public init(recipeId: String)` sets the recipeId and starts loading.
