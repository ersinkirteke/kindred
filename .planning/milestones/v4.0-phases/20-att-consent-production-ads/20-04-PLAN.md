case .consent(.consentFlowCompleted(let status)):

**Why this works:** The `.run` effect accesses the `adClient` dependency via closure capture and calls `configurePersonalization` asynchronously on the main actor (AdMob SDK requires main thread). The `configurePersonalization` closure (implemented in Plan 20-02) sets `GADMobileAds.sharedInstance().requestConfiguration.setPublisherPrivacyPersonalizationState(.allowed)` for `.fullyGranted`, and `.notAllowed` for denied states.

**Gap 2 Fix — Add subscription checks:**

Update `.checkConsentStatus` handler (lines 356-366):

```swift
case .checkConsentStatus:
    // Only trigger consent for authenticated users who completed onboarding
    guard state.hasCompletedOnboarding else { return .none }
    guard case .authenticated = state.currentAuthState else { return .none }

    // Skip consent for pro subscribers (no ads = no consent needed)
    if case .pro = state.profileState.subscriptionStatus {
        return .none
    }

    // Check ATT status
    let attStatus = consentClient.checkATTStatus()
    guard attStatus == .notDetermined else { return .none }

    state.needsConsentFlow = true
    return .send(.consent(.checkConsentOnLaunch))
```

Update `.triggerConsentFlow` handler (lines 368-378):

```swift
case .triggerConsentFlow:
    // Trigger consent flow after onboarding for new users
    guard case .authenticated = state.currentAuthState else { return .none }

    // Skip consent for pro subscribers
    if case .pro = state.profileState.subscriptionStatus {
        return .none
    }

    // Check ATT status
    let attStatus = consentClient.checkATTStatus()
    guard attStatus == .notDetermined else { return .none }

    state.needsConsentFlow = true
    return .send(.consent(.checkConsentOnLaunch))
```

**Why this works:** `SubscriptionStatus.pro` case match exits early if user is a pro subscriber. Pro subscribers never see ads (handled by `FeedReducer.shouldShowAds`), so consent is unnecessary. This matches user decision from Phase 20 CONTEXT.md: "Skip consent flow entirely for pro subscribers (no ads = no need for consent)."

**Edge case handled:** If pro user downgrades to free, the subscription status change will trigger `.checkConsentStatus` again on next launch, showing consent flow at that time.
