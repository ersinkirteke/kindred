/// Combined consent status from UMP (GDPR/CCPA) and ATT (iOS tracking)
public enum ConsentStatus: Equatable, Sendable {
    case notDetermined
    case fullyGranted      // ATT authorized + UMP obtained (or UMP not required)
    case attDenied         // ATT denied, UMP obtained or not required
    case umpDenied         // UMP denied, ATT authorized (rare — GDPR region + ATT allowed)
    case bothDenied        // Both ATT denied and UMP denied
}

/// Internal state for consent flow progression

    case checkingUMP
    case showingUMPForm
    case showingPrePrompt
    case requestingATT
    case completed
}

**1. Create ConsentReducer.swift** at `MonetizationFeature/Sources/Consent/ConsentReducer.swift`:

State machine managing UMP → pre-prompt → ATT flow. Follow TCA @Reducer pattern matching existing SubscriptionReducer.

```swift
@Reducer
public struct ConsentReducer {
    @ObservableState
    public struct State: Equatable {
        public var consentStatus: ConsentStatus = .notDetermined
        public var isShowingPrePrompt: Bool = false
        public var flowStep: ConsentFlowStep = .idle
        public var lastError: String?

        public init() {}
    }

    public enum Action: Equatable {
        // Launch check
        case checkConsentOnLaunch
        case attStatusChecked(isAlreadyDetermined: Bool, isAuthorized: Bool)

        // UMP flow
        case umpConsentCheckCompleted(formAvailable: Bool)
        case umpConsentCheckFailed(String)
        case umpFormPresented
        case umpFormCompleted
        case umpFormFailed(String)

        // Pre-prompt flow
        case showPrePromptIfNeeded
        case prePromptContinueTapped

        // ATT flow
        case attAuthorizationReceived(isAuthorized: Bool)

        // Final resolution
        case consentFlowCompleted(ConsentStatus)

        // Analytics events
        case logAnalyticsEvent(String)
    }
}
```

Reducer body logic:

**`.checkConsentOnLaunch`:**
1. Check `consentClient.checkATTStatus()`
2. If `.authorized` → set state to `.fullyGranted`, send `.consentFlowCompleted(.fullyGranted)`
3. If `.denied` or `.restricted` → set state to `.attDenied`, send `.consentFlowCompleted(.attDenied)`
4. If `.notDetermined` → start UMP check: set flowStep = `.checkingUMP`, run effect calling `consentClient.requestUMPConsentUpdate()`

**`.umpConsentCheckCompleted(formAvailable:)`:**
1. If formAvailable → set flowStep = `.showingUMPForm`, run effect calling `consentClient.presentUMPForm()`, on success send `.umpFormCompleted`, on error send `.umpFormFailed`
2. If !formAvailable → send `.showPrePromptIfNeeded` (no UMP needed in this region)

**`.umpConsentCheckFailed`:**
Log error via OSLog, send `.showPrePromptIfNeeded` (UMP failure does NOT block ATT per user decision)

**`.umpFormCompleted`:**
Send `.showPrePromptIfNeeded`

**`.umpFormFailed`:**
Log error via OSLog, send `.showPrePromptIfNeeded` (continue to ATT regardless)

**`.showPrePromptIfNeeded`:**
1. Check `consentClient.hasSeenPrePrompt()`
2. If already seen → skip to ATT directly: send `.prePromptContinueTapped`
3. If not seen → set `isShowingPrePrompt = true`, flowStep = `.showingPrePrompt`
4. Send `.logAnalyticsEvent("consent_att_shown")`

**`.prePromptContinueTapped`:**
1. Set `isShowingPrePrompt = false`
2. Run effect: `consentClient.markPrePromptSeen()`, then `let status = await consentClient.requestATTAuthorization()`
3. Set flowStep = `.requestingATT`
4. Send `.attAuthorizationReceived(isAuthorized: status == .authorized)`

**`.attAuthorizationReceived(isAuthorized:)`:**
1. Combine UMP + ATT status:
   - Get UMP consent status via `consentClient.getUMPConsentStatus()` (3 = obtained per UMP SDK)
   - If ATT authorized && (UMP obtained or UMP not required) → `.fullyGranted`
   - If ATT authorized && UMP denied → `.umpDenied`
   - If ATT denied && UMP obtained → `.attDenied`
   - If ATT denied && UMP denied → `.bothDenied`
2. Send `.consentFlowCompleted(finalStatus)`
3. Send analytics: `.logAnalyticsEvent("consent_att_authorized")` or `"consent_att_denied"`

**`.consentFlowCompleted(status)`:**
1. Set `consentStatus = status`
2. Set flowStep = `.completed`
3. Run effect: If status is `.attDenied` or `.bothDenied`, call `consentClient.setFirebaseAnalyticsEnabled(false)`. Otherwise call `consentClient.setFirebaseAnalyticsEnabled(true)`.

**`.logAnalyticsEvent`:**
Log via OSLog `Logger(subsystem: "com.ersinkirteke.kindred", category: "consent")`. Firebase event logging handled by the Firebase Analytics integration (existing). For now just log locally — Firebase events will be added when Firebase is properly configured.
